# Biz Skill — 사업 전략 패턴 매칭 엔진

사업 상황 입력 → 18축 성공·실패 패턴 매칭 → 인과 서사로 조건부 전략 도출.

---

## ⛔ 절대 규칙

| # | 규칙 | 이유 |
|---|---|---|
| 1 | **사전식 나열 금지** — 상황→패턴→인과→전략의 서사로 풀 것 | 나열은 판단을 돕지 못함 |
| 2 | **전략 = 상황 조건부** — 유효 조건 + 무효 조건 필수 병기 | 같은 전략도 상황 다르면 독 |
| 3 | **대조군 필수** — 성공 패턴에 같은 조건의 실패 패턴 병치 | 실패가 전략의 경계를 보여줌 |
| 4 | **스포크 로드 필수** — 기억으로 실행 금지, `@uses` + 스크리닝 스포크 로드 | 내재 지식 < 패턴 라이브러리 |
| 5 | **로딩 상한** — `@uses` 3개 상시 + 스크리닝 후 스포크 최대 4개. 초과 시 축 재선별 | 18개 전수 = 토큰 낭비 |

## §3. 실행 파이프라인

공통: **1차 스크리닝**(●핵심/○관련/-무관) → 핵심 축만 스포크 로드(최대 4개) → 모드별 단계 실행.

각 모드의 ①~⑧ 상세 단계·입력·출력 포맷은 분리 문서로.

→ 상세: `references/execution-pipeline.md` 참조

## §4. 서사 원칙

상황→패턴→인과→전략 순서 필수. 전략부터 나열 금지. 6원칙: 인과체인·경계조건·대조군 병치·추상→구체 순서·불확실성 표기.

→ 서사 구조 템플릿: `references/narrative-template.md` 참조

---

## §4.5 리포트 변환 (후처리)

§3 분석 완료 후, 결과를 **그대로 출력하지 않는다.** 리포트 템플릿에 맞춰 재구성. 분석(§3) vs 표현(§4.5) 역할 분리.

**변환 규칙:**
1. §3 실행 과정(스크리닝·로딩·매칭)은 본문 비노출 — 부록 이동
2. 본문은 3블록 구조(진단 서사 → 패턴 대조 서사 → 전략+액션)
3. 각 블록 **최소 3문장 서술형** — 표·리스트 단독 블록 금지
4. **패턴·전략·선택지 소제목·bold·번호 분리 금지** — 같은 문단에서 "반면" "그러나" 접속사로 대비. 판정 모드 A vs B도 예외 없음

→ 리포트 구조 템플릿: `references/report-template.md` 참조

## §6. 매칭 정밀도

**유사도 5변수:** 업종·단계·규모·핵심변수·환경조건

| 등급 | 기준 | 처리 |
|---|---|---|
| A | 4개+ 유사 | 전략 직접 제시 |
| B+ | 3개 유사 (핵심변수 포함) | "조정 필요" 명시 + 전략 제시 가능 |
| B- | 2개 유사 또는 핵심변수 미포함 3개 | "불확실성 높음" 경고 + 판정 모드 시 시나리오 분석 자동 전환 |
| C | 1개 이하 | "참고용만" 경고. 판정 모드 시 시나리오 분석 필수 |

## TURBO 모드 — 고품질 병렬 가속화

**목적:** 전략 진단·설계 시간 단축. 절대규칙·서사 원칙·매칭 정밀도 불변, 스포크 로드·리포트 블록 드래프트만 Agent 분산.

**트리거:** "터보로 전략" · "TURBO" · "비즈스킬 터보" 명시. 미명시 = 일반.

**원칙 (4):**
1. **병렬화만, 스킵 금지** — §3 스크리닝·§5 축간 연결·§6 매칭 정밀도 불변
2. **독립성 확인** — 스크리닝 후 선택된 스포크는 상호 독립. Agent 분산 가능
3. **통합 시퀀셜** — 서사(상황→패턴→인과→전략) 일관성은 메인이 통합
4. **Agent 브리프 완결성** — 축 정의·유사도 5변수·대조군 필수·조건부 전략 템플릿 전달

**병렬 타겟:**
- §3 스크리닝 후 선택된 스포크(최대 4개) 로드·성공/실패 패턴 매칭 — 스포크마다 Agent 1개
- §4.5 리포트 3블록(진단 서사·패턴 대조 서사·전략+액션) 드래프트 — 블록마다 Agent 1개

**병렬 제외 (시퀀셜):**
- §2 모드 판별 (진단/전략/판정 확정)
- §3 1차 스크리닝 (18축 우선순위) — 선행 판정
- §5 축간 연결 맵 적용 (선행 축 결정이 후행 축에 영향 → 순차)
- §6 매칭 정밀도 판정 (A/B+/B-/C 등급)
- 최종 리포트 통합·서사 일관성 검증·대조군 병치 확인

**품질 저하 방어:**
- 병렬 결과 교차검증 — 유효/무효 조건 충돌·대조군 누락 검출
- 서사 접속사 손상(소제목·bold 분리) 감지 → 해당 블록 시퀀셜 재작성
- 성공 패턴만 나열·단정 표현 감지 → 경고 후 재작성

**예상 단축:** 40~55% (스포크 수·리포트 분량에 비례)
