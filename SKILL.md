## ⛔ 절대 규칙

#규칙이유1**사전식 나열 금지** — 상황→패턴→인과→전략의 서사로 풀 것나열은 판단을 돕지 못함2**전략 = 상황 조건부** — 유효 조건 + 무효 조건 필수 병기같은 전략도 상황 다르면 독3**대조군 필수** — 성공 패턴에 같은 조건의 실패 패턴 병치실패가 전략의 경계를 보여줌4**스포크 로드 필수** — 기억으로 실행 금지, uses + 스크리닝 스포크 로드내재 지식 &lt; 패턴 라이브러리5**로딩 상한** — uses 3개 상시 + 스크리닝 후 스포크 최대 4개. 초과 시 축 재선별18개 전수 = 토큰 낭비

## §4.5 리포트 변환 (후처리)

§3 분석 완료 후, 결과를 **그대로 출력하지 않는다.** 리포트 템플릿에 맞춰 재구성. 분석(§3) vs 표현(§4.5) 역할 분리.

**변환 규칙:**

1. §3 실행 과정(스크리닝·로딩·매칭)은 본문 비노출 — 부록 이동
2. 본문은 3블록 구조(진단 서사 → 패턴 대조 서사 → 전략+액션)
3. 각 블록 **최소 3문장 서술형** — 표·리스트 단독 블록 금지
4. **패턴·전략·선택지 소제목·bold·번호 분리 금지** — 같은 문단에서 "반면" "그러나" 접속사로 대비

## 📄 표준 리포트 템플릿 (McKinsey SCR 구조)

**§4.5 리포트 변환 시 아래 템플릿을 사용한다. 블록을 표·리스트로만 채우는 것 금지 — 각 블록 최소 3문장 서술.**

### 전략 진단 리포트 (진단/전략 모드)

```markdown
# 전략 진단: [사업/프로젝트명]
작성일: | 모드: 진단/전략/판정 | 매칭 등급: A/B+/B-/C

## Executive Summary (SCR)
**Situation:** [현재 상황 — 단계·규모·핵심 지표 포함 1~2문장]
**Complication:** [왜 지금 이 전략 판단이 중요한가 — 위기/기회/분기점 1~2문장]
**Resolution:** [핵심 전략 권고 — 조건 포함 1~2문장]

## 1. 진단 서사 (상황→패턴)
[현재 상황을 18축 중 핵심 축과 연결하여 서술. 어떤 패턴이 작동하고 있는지 인과체인으로 설명.
성공 패턴만 나열하지 않고, 동일 조건에서의 실패 패턴도 대비. 최소 3문장.]

## 2. 패턴 대조 서사 (성공 vs 실패)
[같은 상황에서 성공한 사례와 실패한 사례를 병치하여 경계 조건 도출.
유효 조건과 무효 조건을 서사 안에서 명시. 최소 3문장.]

## 3. 전략 + 액션 (조건부)
[전략 권고를 조건부로 서술. 추상 전략이 아닌 구체 액션으로 착지. 최소 3문장.]

### 실행 액션
| 구분 | 액션 | 기한 | 선행 조건 |
|------|------|------|---------|
| 즉시 (D+30) | | | |
| 단기 (D+90) | | | |
| 전략 (6개월+) | | | |

확신도: [90/70/50/30] — [근거: 매칭 등급 + 유사도 변수]
```

### 판정 리포트 (A vs B)

```markdown
# 전략 판정: [선택지 A] vs [선택지 B]
작성일: | 매칭 등급:

## 판정 결론
> [A를 선택한다 / B를 선택한다 / 조건부 A] — 이유 1문장

## 상황 공통 맥락
[A와 B 모두에게 적용되는 현재 상황 — 2~3문장]

## A 분석
[A를 선택했을 때의 패턴·인과·리스크. 유효 조건과 무효 조건 병기. 최소 3문장.]

유효 조건: [A가 작동하는 전제]
무효 조건: [A가 실패하는 상황]

## B 분석
[B를 선택했을 때의 패턴·인과·리스크. 최소 3문장.]

유효 조건: [B가 작동하는 전제]
무효 조건: [B가 실패하는 상황]

## 판정 근거
[A와 B를 직접 대조. 현재 상황 조건에서 어떤 패턴이 더 유리한지. 최소 3문장.]

권고: [A / B / 조건부 A (X 조건 충족 시)]
이 판정이 바뀌는 조건: [무엇이 달라지면 반대 선택이 유리한가]
```

## PRE_WRITE — 진단·전략·판정 리포트 작성 직전 룰

**목적:** SCR·서사·판정 *작성 시점*에 단언형·수치+출처·조건부 강제. §4.5 리포트 변환의 사후 + PRE_WRITE 사전 = 이중 게이트.

### 5룰 (작성 직전 강제)

| # | 룰 | 전략 결로 변환 | FAIL 신호 |
|---|---|---|---|
| 1 | **SCR 단문** | Situation·Complication·Resolution 각 1~2문장 ≤50자 | 80자+·중첩 |
| 2 | **단일 권고** | Resolution = 1전략 (조건부 OK). 다중 ✗ | "A도 B도 고려" |
| 3 | **수치+출처** | 모든 패턴·전략 = 수치 + 사례 출처 1조 | "성공한 사례가 많다" 추상 |
| 4 | **조건부 명시** | "유효 조건·무효 조건" verbatim 명시 | 무조건적 단언 |
| 5 | **AI식 사전회피** | "전략적 우위", "지속 가능한 성장", "차별화된 접근" — 작성 단계 차단 | 컨설팅 관용어 |

### 모드별 PRE_WRITE

- **진단**: SCR 1~2문장 ≤50자. 매칭 등급 단언
- **전략**: 권고 = 조건부 1전략. 액션 동사시작
- **판정 (A vs B)**: 결론 1문장. 유효·무효 조건 verbatim

### 자체검증

| # | 체크 | 위반 |
|---|------|------|
| 1 | SCR 각 ≤50자? | 초과 = 압축 |
| 2 | 조건부 명시? | 무조건 단언 = 조건 추가 |
| 3 | 수치+출처? | 누락 = 사례 보완 |
| 4 | AI식 컨설팅 관용어 hit? | hit ≥ 1 = 평문 변환 |

## §CONFIRM_GATE — 송출 직전 형 컨펌 (3단계 가드)

```
🔍 송출 전 검토 부탁드려요. AI 티·번역투·장황 있나요?

```

**규칙:**

**SCOPE_OUT (컨펌 면제):**
