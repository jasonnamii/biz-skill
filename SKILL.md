# Biz Skill — 사업 전략 패턴 매칭 엔진

사업 상황 입력 → 18축 성공·실패 패턴 매칭 → 인과 서사로 조건부 전략 도출.

---

## ⛔ 절대 규칙

| # | 규칙 | 이유 |
|---|---|---|
| 1 | **사전식 나열 금지** — 상황→패턴→인과→전략의 서사로 풀 것 | 나열은 판단을 돕지 못함 |
| 2 | **전략 = 상황 조건부** — 유효 조건 + 무효 조건 필수 병기 | 같은 전략도 상황 다르면 독 |
| 3 | **대조군 필수** — 성공 패턴에 같은 조건의 실패 패턴 병치 | 실패가 전략의 경계를 보여줌 |
| 4 | **스포크 로드 필수** — 기억으로 실행 금지, `@uses` 로드 | 내재 지식 < 패턴 라이브러리 |
| 5 | **전수 스캔 금지** — 스크리닝 후 핵심 축만 로드 (최대 4개) | 18개 전수 = 토큰 낭비 |

## §3. 실행 파이프라인

### 공통: 1차 스크리닝

18축을 ●핵심/○관련/-무관으로 1줄 판정 → ● 축만 스포크 로드 (최대 4개).

### 스포크 로딩 — Python 인덱서 우선

스포크 로드 시 Python 파서로 1회 전체 인덱싱 후, 줄번호 기반 정독을 진행한다.

**Step 1: 전체 인덱스 1회 출력**

```bash
python scripts/spoke_loader.py index-all references/
```

→ 18개 스포크의 패턴명+줄번호를 1회로 확보. 별도 Read 불필요.

**Step 2: 매칭 후보만 줄번호로 정독**

```bash
python scripts/spoke_loader.py read-lines references/g2-growth.md 45 92
```

→ 해당 패턴 섹션만 추출. 전량 로드 방지.

**폴백 (Python 불가 시)**

전통 2단계 로딩:
```bash
python scripts/spoke_loader.py scan references/f1-market.md
python scripts/spoke_loader.py read references/f1-market.md "패턴1" "실패패턴2"
```

**로딩 검증 체크**: ①인덱싱 단계에서 매칭 후보 패턴 3개+ 식별 필수 (3개 미만 시 인접 스포크 확장 탐색) ②정독 후 "매칭 근거 1줄" 필수 기록 (근거 없는 매칭 = 무효) ③후보 0건 시 → 해당 축은 "매칭 불가"로 판정, 판정 모드에서는 시나리오 분석 전환(§3 ⑧) 트리거

### 모드 공통 ②단계

모든 모드의 ②는 동일: **1차 스크리닝 → 핵심 축 식별 → 스포크 로드** (위 2단계 로딩 적용)

### 진단 모드

①상황 구조화 — 업종·단계·규모·핵심 이슈·제약 조건 추출
②스크리닝→스포크 로드
③패턴 매칭 — 유사도 기준(업종·단계·제약·핵심변수)으로 성공·실패 패턴 식별
④서사형 진단:
  "당신의 상황은 [패턴 A]와 유사합니다. [패턴 A]에서는 [조건 X]일 때 [결과 Y]가 발생했습니다. 현재 [조건 X']가 [유사/다른] 점은..."
⑤대조군 병치 — 같은 축에서 반대 결과를 낳은 패턴: "같은 조건에서 [패턴 B]는 실패. 차이점은 [변수 Z]."
⑥축간 영향 맵 — 핵심 축 상태가 다른 축에 미치는 영향

**출력**: **1차 스크리닝(●/○/-)** → 상황 구조화 → 패턴 매칭 서사 → 대조군 → 강점/약점 → 축간 영향

### 전략 모드

①목표 명확화 — 모호하면 형에게 확인
②스크리닝→스포크 로드
③상황-패턴 매칭 — "이 상황에서 이 목표를 달성한 패턴"이 핵심
④조건부 전략 도출 — 반드시 아래 형식:

```
전략: [구체적 전략]
유효 조건: [이 전략이 작동하는 상황]
무효 조건: [이 전략이 역효과 나는 상황]
근거 패턴: [어떤 상황에서 이 전략이 작동했는가]
대조 패턴: [비슷한 전략이 실패한 케이스]
```

⑤전략 간 우선순위 — 임팩트·실행 가능성 기준
⑥리스크 체크 — 각 전략의 실패 시나리오를 대조군에서 추출
⑦액션 플랜 — 전략 → 구체적 실행 단계로 번역

**출력**: **1차 스크리닝(●/○/-)** → 목표 확인 → 상황-패턴 매칭 서사 → 조건부 전략(유효/무효) → 대조군 리스크 → **축간 영향 체크** → 액션 플랜

### 판정 모드

①갈림길 구조화 — 선택지 A vs B (vs C)의 전제·기대 결과·우려 정리
②스크리닝→스포크 로드
③양쪽 패턴 매칭 — 같은 갈림길에서 다른 선택을 한 사례 쌍이 가장 강력
④판단 기준 — 양쪽 성패를 가른 핵심 변수: "A 성공 조건: [X]. B 성공 조건: [Y]. 당신의 상황에서 [X]는 [충족/미충족]."
⑤리스크 매트릭스: | 선택지 | 최선 | 최악 | 핵심 변수 | 되돌림 가능성 |
⑥판정 보조 — 데이터가 가리키면 방향 제시, 불충분하면 "필요한 추가 정보" 명시
⑦**재평가 기한 필수** — 재평가 시점(3/6/12개월 택1) + 재평가 트리거 변수 1개+ 명시. 불확실성 하의 판정이 고정되지 않도록 사전 설계
⑧**시나리오 분석 전환** — 매칭등급 B- 이하(§6) → 패턴 매칭 대신 3시나리오(낙관/기본/비관) 분석 자동 전환. 각 시나리오별 핵심 변수 상태 + 예상 결과 + 확률 가중치(`[전문가판단]` 태그) + "패턴 매칭 불충분" 명시

**출력**: **1차 스크리닝(●/○/-)** → 갈림길 구조화 → 양쪽 패턴 서사 → 판단 기준 → 리스크 매트릭스 → **축간 영향 체크** → 판정 보조 → **재평가 기한** → (B- 이하 시) 시나리오 분석

## Gotchas

1. 패턴 사전식 나열 ✗ → 상황→매칭→인과 서사→조건부 전략
2. "이러면 성공" 단정 ✗ → 유효·무효 조건 + 대조군 필수
3. 성공 패턴만 ✗ → 대조군 없는 전략 = 생존자 편향
4. 18개 전수 로드 ✗ → 스크리닝 후 최대 4개
5. 경계 조건 생략 ✗ → 작동 범위 명시
6. 추상 전략만 ✗ → 액션 플랜 구체화
7. 축간 영향 무시 ✗ → §5 체크 필수
8. C등급인데 단정 ✗ → "참고용" 명시
