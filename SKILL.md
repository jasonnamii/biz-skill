# Biz Skill — 사업 전략 패턴 매칭 엔진

사업 상황 입력 → 18축 성공·실패 패턴 매칭 → 인과 서사로 조건부 전략 도출.

---

## ⛔ 절대 규칙

| # | 규칙 | 이유 |
|---|---|---|
| 1 | **사전식 나열 금지** — 상황→패턴→인과→전략의 서사로 풀 것 | 나열은 판단을 돕지 못함 |
| 2 | **전략 = 상황 조건부** — 유효 조건 + 무효 조건 필수 병기 | 같은 전략도 상황 다르면 독 |
| 3 | **대조군 필수** — 성공 패턴에 같은 조건의 실패 패턴 병치 | 실패가 전략의 경계를 보여줌 |
| 4 | **스포크 로드 필수** — 기억으로 실행 금지, `@uses` 로드 | 내재 지식 < 패턴 라이브러리 |
| 5 | **전수 스캔 금지** — 스크리닝 후 핵심 축만 로드 (최대 4개) | 18개 전수 = 토큰 낭비 |

## §3. 실행 파이프라인

### 공통: 1차 스크리닝

18축을 ●핵심/○관련/-무관으로 1줄 판정 → ● 축만 스포크 로드 (최대 4개).

### 스포크 단계적 로딩

스포크 로드 시 전량 로드가 아닌 2단계 로딩을 적용한다.

| 단계 | 로드 범위 | 목적 |
|---|---|---|
| **1단계: 스캔** | 각 스포크의 `## 성공 패턴 라이브러리`·`## 실패 패턴 라이브러리` 헤딩 + `### 패턴명` 목록만 | 어떤 패턴이 매칭 후보인지 빠르게 식별 |
| **2단계: 정독** | 매칭 후보 패턴의 전체 내용 (작동조건·인과·경계·사례·대조) | 서사 도출에 필요한 디테일 확보 |

**이유:** 거대 스포크(e5: 790줄, g2: 931줄, s3: 748줄)를 전량 로드하면 단일 스포크가 컨텍스트의 상당 부분을 점유한다. 1단계 스캔으로 불필요한 패턴을 걸러낸 후, 필요한 패턴만 정독하면 토큰 효율이 2~3배 개선된다.

**로딩 검증 체크**: ①스캔 단계에서 매칭 후보 패턴 3개+ 식별 필수 (3개 미만 시 인접 스포크 확장 탐색) ②정독 후 "매칭 근거 1줄" 필수 기록 (근거 없는 매칭 = 무효) ③후보 0건 시 → 해당 축은 "매칭 불가"로 판정, 판정 모드에서는 시나리오 분석 전환(§3 ⑧) 트리거

**Python 파서 (선택):** `scripts/spoke_loader.py scan|read|headings {파일} [패턴명...]`

### 모드 공통 ②단계

모든 모드의 ②는 동일: **1차 스크리닝 → 핵심 축 식별 → 스포크 로드** (위 2단계 로딩 적용)

### 진단 모드

①상황 구조화 — 업종·단계·규모·핵심 이슈·제약 조건 추출
②스크리닝→스포크 로드
③패턴 매칭 — 유사도 기준(업종·단계·제약·핵심변수)으로 성공·실패 패턴 식별
④서사형 진단:
  "당신의 상황은 [패턴 A]와 유사합니다. [패턴 A]에서는 [조건 X]일 때 [결과 Y]가 발생했습니다. 현재 [조건 X']가 [유사/다른] 점은..."
⑤대조군 병치 — 같은 축에서 반대 결과를 낳은 패턴: "같은 조건에서 [패턴 B]는 실패. 차이점은 [변수 Z]."
⑥축간 영향 맵 — 핵심 축 상태가 다른 축에 미치는 영향

**출력**: **1차 스크리닝(●/○/-)** → 상황 구조화 → 패턴 매칭 서사 → 대조군 → 강점/약점 → 축간 영향

### 전략 모드

①목표 명확화 — 모호하면 형에게 확인
②스크리닝→스포크 로드
③상황-패턴 매칭 — "이 상황에서 이 목표를 달성한 패턴"이 핵심
④조건부 전략 도출 — 반드시 아래 형식:

```
전략: [구체적 전략]
유효 조건: [이 전략이 작동하는 상황]
무효 조건: [이 전략이 역효과 나는 상황]
근거 패턴: [어떤 상황에서 이 전략이 작동했는가]
대조 패턴: [비슷한 전략이 실패한 케이스]
```

⑤전략 간 우선순위 — 임팩트·실행 가능성 기준
⑥리스크 체크 — 각 전략의 실패 시나리오를 대조군에서 추출
⑦액션 플랜 — 전략 → 구체적 실행 단계로 번역

**출력**: **1차 스크리닝(●/○/-)** → 목표 확인 → 상황-패턴 매칭 서사 → 조건부 전략(유효/무효) → 대조군 리스크 → **축간 영향 체크** → 액션 플랜

### 판정 모드

①갈림길 구조화 — 선택지 A vs B (vs C)의 전제·기대 결과·우려 정리
②스크리닝→스포크 로드
③양쪽 패턴 매칭 — 같은 갈림길에서 다른 선택을 한 사례 쌍이 가장 강력
④판단 기준 — 양쪽 성패를 가른 핵심 변수: "A 성공 조건: [X]. B 성공 조건: [Y]. 당신의 상황에서 [X]는 [충족/미충족]."
⑤리스크 매트릭스: | 선택지 | 최선 | 최악 | 핵심 변수 | 되돌림 가능성 |
⑥판정 보조 — 데이터가 가리키면 방향 제시, 불충분하면 "필요한 추가 정보" 명시
⑦**재평가 기한 필수** — 재평가 시점(3/6/12개월 택1) + 재평가 트리거 변수 1개+ 명시. 불확실성 하의 판정이 고정되지 않도록 사전 설계
⑧**시나리오 분석 전환** — 매칭등급 B- 이하(§6) → 패턴 매칭 대신 3시나리오(낙관/기본/비관) 분석 자동 전환. 각 시나리오별 핵심 변수 상태 + 예상 결과 + 확률 가중치(`[전문가판단]` 태그) + "패턴 매칭 불충분" 명시

**출력**: **1차 스크리닝(●/○/-)** → 갈림길 구조화 → 양쪽 패턴 서사 → 판단 기준 → 리스크 매트릭스 → **축간 영향 체크** → 판정 보조 → **재평가 기한** → (B- 이하 시) 시나리오 분석

## §4. 서사 원칙 — 산출물의 뼈대

사전식 나열이 아닌 서사형 산출물을 보장하기 위한 원칙.

| # | 원칙 |
|---|---|
| 1 | **상황 먼저, 전략 나중** — 상황 구조화→패턴 발견→인과 후에야 전략. 전략부터 나열 금지 |
| 2 | **인과 체인 필수** — "A→B"가 아니라 "[조건 C] 하에서 [메커니즘 D]가 작동하여 B" |
| 3 | **경계 조건 명시** — 패턴 작동 범위 밖에서는 반대 결과. 경계를 명시 |
| 4 | **대조군 = 서사의 쌍둥이** — 성공 서사에 실패 서사가 자동으로 따라옴. 단독 금지 |
| 5 | **추상→구체** — 패턴(추상) 먼저, 형의 상황(구체) 매핑 다음. 역순 금지 |
| 6 | **불확실성 표기** — 매칭 확신 낮으면 "적용 주의" 명시. 확신 없는 단정 금지 |

### 서사 구조 템플릿

```
[상황 읽기]
  형의 상황을 핵심 변수 중심으로 구조화

[패턴 발견]
  "이 상황은 [패턴명]과 구조적으로 유사합니다."
  — 유사점: ...
  — 차이점: ...

[패턴의 인과]
  "[패턴명]에서는 [조건] 하에서 [행동]이 [메커니즘]을 통해 [결과]를 만들었습니다."

[대조군]
  "반면 [대조 패턴]에서는 같은 [조건]에서 [다른 행동/미묘한 차이]가 [반대 결과]를 낳았습니다."
  — 성패를 가른 변수: ...

[적용]
  "형의 상황에서 이 패턴을 적용하면..."
  — 유효 조건: ...
  — 주의 조건: ...

[전략]
  위 맥락 위에서 비로소 구체적 전략 제시
```

## §5. 축간 연결 맵

전략 도출 시 반드시 참조. 한 축의 판단이 다른 축에 미치는 영향.

| 선행 | → 후행 | 메커니즘 |
|---|---|---|
| F1 | → G1, F2 | 시장→PMF 전제, 고객→BM 제약 |
| F2→E2 | F3→G3 | BM→펀딩, 해자 없는 스케일=경쟁자 초대 |
| F4→F3 | G1→G2 | 기술→해자, 0→1→성장 전제 |
| G2 | → E2, E1 | 성장→펀딩·조직 확장 압력 |
| G3→E1 | G5→F3 | 스케일업 제약=조직, NW효과=최강 해자 |
| S1→F1 | S2→G4 | 피봇=시장 재선택, 파트너=국제화 |
| S3→E1 | E1→전체 | 다각화→복잡도↑, 실행력의 기반 |
| E2→E3 | E3→S1,S4 | 자본 고갈=생존 위기→피봇/퇴장 |
| E4→F1 | E5→S1,G1 | 규제→시장 변경, 타이밍→PMF |

## §6. 매칭 정밀도

**유사도 5변수**: 업종·단계·규모·핵심변수·환경조건

| 등급 | 기준 | 처리 |
|---|---|---|
| A | 4개+ 유사 | 전략 직접 제시 |
| B+ | 3개 유사 (핵심변수 포함) | "조정 필요" 명시 + 전략 제시 가능 |
| B- | 2개 유사 또는 핵심변수 미포함 3개 | "불확실성 높음" 경고 + 판정 모드 시 **시나리오 분석 자동 전환**(§3 ⑧) |
| C | 1개 이하 | "참고용만" 경고. 판정 모드 시 시나리오 분석 필수 |

## §7. 스포크 표준 구조

모든 스포크는 아래 섹션을 순서대로 포함해야 한다. 누락 시 해당 스포크의 산출물 품질이 저하된다.

| # | 섹션 | 필수 | 비고 |
|---|---|---|---|
| 1 | `## 성공 패턴 라이브러리` | ✓ | 각 패턴: 작동조건→인과메커니즘→경계조건→사례→대조사례 |
| 2 | `## 실패 패턴 라이브러리` | ✓ | 성공과 별도로 실패 고유 패턴 정리 |
| 3 | `## 패턴 쌍 대조표` | ✓ | 성공↔실패를 같은 조건 기준으로 병치. **이것이 절대규칙 #3의 구현체** |
| 4 | `## 진단 체크리스트` | ✓ | 해당 축의 현 상태를 빠르게 판정하는 질문 목록 |
| 5 | `## 조건부 전략 매트릭스` | ✓ | 상황 변수 조합별 전략 분기 |
| 6 | `## 축간 연결` | ✓ | §5 연결맵의 해당 축 상세 |
| 7 | 참고 자료 | 선택 | 출처·레퍼런스 |

섹션 명칭 소폭 변형은 허용. 섹션 자체의 부재는 불허.

## Gotchas

1. 패턴 사전식 나열 ✗ → 상황→매칭→인과 서사→조건부 전략
2. "이러면 성공" 단정 ✗ → 유효·무효 조건 + 대조군 필수
3. 성공 패턴만 ✗ → 대조군 없는 전략 = 생존자 편향
4. 18개 전수 로드 ✗ → 스크리닝 후 최대 4개
5. 경계 조건 생략 ✗ → 작동 범위 명시
6. 추상 전략만 ✗ → 액션 플랜 구체화
7. 축간 영향 무시 ✗ → §5 체크 필수
8. C등급인데 단정 ✗ → "참고용" 명시
