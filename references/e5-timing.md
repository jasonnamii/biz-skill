# 타이밍·거시환경 — 시점과 환경을 어떻게 읽는가?

## 핵심 질문
**"시점과 환경을 어떻게 읽는가?"**

성공하는 창업자는 최적의 시점을 어떻게 인식하는가? 거시경제 순환, 기술 도입곡선, 시장 전환이 기회의 창을 어떻게 만드는가? 너무 일찍 진입할 때의 비용 대 너무 늦게 진입할 때의 비용은?

---

## 핵심 명제

타이밍은 **스타트업 성공과 실패의 차이의 42%를 차지한다**(Bill Gross, 100+ 회사 분석, 2016). 그러나 타이밍은 무작위 운이 아니다. 학습 가능한 기술이다: (1) 거시경제 순환과 자본 가용성 읽기, (2) 기술 도입곡선을 추적하여 변곡점 식별, (3) 소비자 행동이 새로운 비즈니스 모델을 지지할 만큼 충분히 변했을 때 인식.

성공은 첫 번째가 아니라 **시장이 준비되었을 때 진입하는 것**에서 온다. 데이터:
- **성공 요인의 42%**: 타이밍 (Bill Gross)
- **첫 번째 진입자의 47%는 실패** vs. **빠른 추종자의 8% 실패** (Golder & Tellis, 1993)
- **Fortune 500의 50%**는 위기나 불황 중에 설립됨
- **S자 곡선 채택 가속**: 변곡점은 과장이 사라지고 헌신적인 실무자들이 실질적 가치를 증명할 때 발생

## 성공 패턴 라이브러리

### 패턴 1: 골디락스 존 — 너무 빠르지도, 너무 늦지도 않게

#### 작동조건
최적 진입 윈도우는 다음과 같을 때 발생:
1. **핵심 기술이 충분히 성숙** (더 이상 베타 단계가 아님)
2. **인프라가 충분함** (기존 업체들에 의해 아직 상품화되지 않음)
3. **소비자 행동이 새로운 모델을 수용하도록 변함**
4. **스케일링을 위한 자본이 가용**

#### 인과메커니즘
이 윈도우 이전 진입 = 고객 저항 + 자본 부족.
이 윈도우 이후 진입 = 네트워크 효과를 가진 견고한 경쟁자.
윈도우 내 진입 = 선택권 우위와 빠른 확장.

#### 경계조건
- 기술이 아직 불안정하면 브랜드 손상
- 시장이 이미 포화되면 네트워크 효과로 인해 승리 불가능
- 자본이 가용하지 않으면 스케일링 불가능

#### 사례: Netflix 스트리밍 (2007)

**타임라인:**
- **2007년 1월 16일**: Netflix "Watch Now" 스트리밍 기능 출시
- **광대역 채택**: 2009년까지 미국 가구의 40%가 광대역 보유 (최소 1 Mbps, DVD 품질은 3 Mbps 필요)
- **콘텐츠 제한**: 1,000개 타이틀 vs. DVD상 70,000개
- **2010년대**: 기술 성숙, 광대역 표준화, 콘텐츠 라이브러리 10배 성장
- **2020년**: 스트리밍이 기본 모델; DVD 렌탈 중단

**작동 이유:** Netflix는 정확한 변곡점 순간에 진입:
- 광대역 인프라 존재하나 경쟁자들의 포화 전
- 기술 안정적 (압축, 스트리밍 프로토콜)이나 최적화 전
- 소비자 행동 변화: 젊은 시청자들 스트리밍 선호도 표시; ISP 속도 개선
- 기존 Blockbuster는 DVD 배포 모델에 갇힘

**핵심 수치:**
- 2012년: 일일 1백만 시간 시청
- 시장 전환: 스트리밍 수익이 약 2015년경 DVD 추월
- 현재 시장점유율: 글로벌 SVOD 시장의 약 30% (2024년)

**교훈**: 시간이 아니라 도입곡선을 봐라. Netflix의 2007년 타이밍은 광대역이 S자 곡선의 **가파른 단계**에 있었기 때문에 작동했다 — 개척하기 충분히 일찍이면서도 인프라가 이미 존재할 만큼 충분히 늦게.

#### 사례: Zoom (2011년 설립 → 2019년 IPO → 2020년 COVID 폭발)

**타임라인:**
- **2011년 4월**: Eric Yuan (전직 Cisco 협업 엔지니어)이 설립
- **2013년 1월**: 첫 버전 출시
- **2013-2019**: 9년간 제품 정제; 완만한 엔터프라이즈 채택
- **2019년 4월**: IPO $36/주; 약 $160억 가치
- **2020년 5월-7월**: COVID-19 락다운; Zoom은 가정용 이름이 됨
  - **2020년 Q2**: 3개월 수익 > 2019년 전체 수익
  - **2020년 순위**: 전 세계 5번째로 가장 많이 다운로드된 모바일 앱 (TikTok, WhatsApp, Facebook, Instagram 뒤)
- **최고 가치**: $1600억 (2020년 12월)

**작동 이유:** Zoom은 **장기 준비가 거시 충격을 만남**:
- **2011-2019**: 아직 대규모로 존재하지 않는 시장에서 제품 구축; 완만한 채택
- **비디오 컨퍼런싱 인프라**: 충분히 성숙 (대역폭, 코덱, IP 표준)이지만 기존자들 (Cisco Webex, Microsoft Teams)은 복잡/비쌈
- **COVID-19 트리거 (2020년 3월)**: 갑자기 수백만 명이 간단한 비디오 컨퍼런싱 필요; 학교와 사무실 완전 원격 전환
- **네트워크 효과**: "사용하기 쉬운" 범주의 첫 진입자; 기본 선택이 됨
- **단위 경제**: 프리미엄 모델; 무료 사용자가 유료 엔터프라이즈 고객이 됨

**핵심 수치:**
- COVID 전 성장: 연 100% (이미 강함)
- COVID 가속: 일일 회의 분 300백만 (2019년 12월) → 33억 (2020년 4월) — 4개월에 11배

**교훈**: 장기 창업자들은 미래를 예측하기 때문이 아니라 **현재 수요를 캡처하면서 미래를 위해 구축**하기 때문에 성공한다.

#### 대조사례: Google Glass (2012 발표 → 2013 익스플로러 출시 → 2015 중단)

**타임라인:**
- **2012년**: Google이 Project Glass 발표; Time Magazine이 "올해의 최고 발명"으로 선정
- **2013년**: 개발자 버전 출시 (~$1,500)
- **2015년**: 제품 중단; 공개 판매에서 제거
- **왜 시기가 틀렸는가**: 2013년 웨어러블 AR 기술은 소비자 규모에서 준비되지 않음

**너무 일찍인 이유:**
1. **기술 미성숙**:
   - 배터리 수명: 약 2-3시간 (하루종일 착용 불가능)
   - 디스플레이 기술: 외안경 투사는 제한적; 대부분의 사용 사례에 충분히 몰입적이지 않음
   - 무게와 편의성: 부자연스러움; 사회적 낙인 ("glassholes")

2. **시장 준비**:
   - 앱 생태계 없음 (2007년 스마트폰과 달리)
   - 비용 ($1,500)은 웨어러블 소비자 기대치를 훨씬 초과
   - 사용 사례 불명확: 전화기를 들고 다니는 것 대신 이것을 왜 착용?
   - 사회적 우려: 개인정보, 주의산만, 규범 — 결코 해결되지 않음

3. **인프라**:
   - 모바일 프로세서는 AR에 최적화되지 않음; 테더링에 의존
   - 무선 네트워크 (LTE) 방금 도착; 5G 없음
   - 컴퓨터 비전 알고리즘 미성숙

**S자 곡선 문제**: Glass는 도입곡선의 **"과장 피크"**에서, **가파른 성장 단계**가 아니라 도착했다. 과장이 효용을 선행했다.

**나중 성공**: 10년 후 (2023-2025), AR 안경 (Meta Quest Pro, Apple Vision Pro)은 다음으로 출시:
- 소비자 가격대 ($499-$3,500, 엔터프라이즈 전용 대신)
- 이미 구축된 앱 생태계
- 더 나은 프로세서 (스마트폰급 AI 칩)
- 게임 덕분에 몰입형 기술의 소비자 기대치

**교훈**: 미디어 과장을 시장 준비와 혼동하지 마라.

### 패턴 2: S자 곡선 탈 때 변곡점 인식

#### 작동조건
**S자 곡선 도입 모델**은 기술이 어떻게 확산되는지 설명:
1. **완만한 단계 (0-20%)**: 초기 채택자; 높은 비용; 제한된 가시성
2. **가파른 단계 (20-80%)**: 빠른 가속; 네트워크 효과 시작; 임계량 달성
3. **포화 단계 (80%+)**: 성장 둔화; 시장 성숙; 경쟁 심화

**핵심 통찰**: 변곡점 (곡선이 가장 가파를 때)은 종종 미디어 과장이 **가장 낮을 때** 발생 — 과장 주기가 붕괴한 후, 헌신적인 실무자들이 실질적 가치를 증명했을 때.

#### 인과메커니즘
시작자가 **가파른 단계**에 진입하면 (과장 피크가 아닌), 시장 규모, 경쟁 강도, 채택 속도를 활용하면서도 기술적 리스크는 낮다.

#### 경계조건
- 너무 일찍: 기술 리스크, 사용 사례 불명확, 도입 느림
- 너무 늦게: 기존 업체들이 네트워크 효과로 장악

#### 사례: Slack (2013 설립 → 2014 공개 베타 → 2019 IPO $195억)

**타임라인:**
- **초기 2013**: Stewart Butterfield가 게임 스타트업 (Tiny Speck)을 직장 커뮤니케이션 도구로 피벗
- **2013년 8월**: Slack 공개 베타 출시
- **첫 주 (2013년 8월)**: 첫 날 8,000팀 가입; 2주일 내 15,000팀
- **2014년 2월**: 공개 출시
- **2019년**: $195억으로 직접 상장 (전통적 IPO 없음)
- **2021년**: 최고 가치 약 $430억 (Salesforce 인수 협상 전)

**타이밍이 완벽했던 이유:**
1. **직장 커뮤니케이션의 S자 곡선 변곡점**:
   - 이메일은 포화 (레거시, 스팸, 깨진 스레딩)
   - 인스턴트 메시징 존재 (AIM, Skype)했으나 소비자 중심
   - Slack은 비즈니스가 비동기, 검색 가능, 조직된 채팅이 필요함을 깨달았을 때 도착
   - 원격근무 추세 가속 시작

2. **기술 준비**:
   - 클라우드 인프라 (AWS) 성숙; 쉬운 확장
   - 모바일 기기 (iOS/Android) 유비쿼터스; 모바일 우선 앱 예상
   - API와 통합이 표준화 중
   - 실시간 메시징 (WebSockets) 해결된 문제

3. **시장 행동 변화**:
   - 밀레니얼/Gen-Z 인력이 노동시장에 진입, 현대적 도구 기대
   - 빠른 비즈니스 순환은 이메일보다 빠른 커뮤니케이션 요구
   - 분산 팀 (초기 원격근무)은 중앙 허브 필요

4. **경쟁 환경**:
   - Microsoft Teams는 2016년까지 존재하지 않음 (Slack 이후 3년)
   - Teams가 도착했을 때, Slack은 이미 네트워크 효과와 팀 전환 비용 보유
   - Facebook Workplace (2016)는 견인력을 얻지 못함

**핵심 수치:**
- 2014년: 첫 달 15,000팀
- 2019년: 500,000+ 유료 계정, 400만+ 일일 활성 사용자
- 일일 메시지량: 2019년까지 하루 10억+ 메시지

**교훈**: Slack은 직장 채팅을 발명하지 않았다; S자 곡선이 가파를 때 진입했다 — 필요가 급할 때, 기술이 준비되었을 때, 고객 행동이 변할 때.

#### 사례: Airbnb (2008 설립 → 2016 IPO → 2023 팬데믹 후 성장)

**타임라인:**
- **2007년 10월**: Brian Chesky가 일을 그만두고 San Francisco로 Joe Gebbia와 함께 이동
- **2008년 여름**: 글로벌 금융 위기; 투자자들이 Airbnb 투자안을 보편적으로 거절
- **2008년 절망적 피벗**: Obama O's와 Cap'n McCain's 시리얼 박스 인쇄; 800개 판매로 $30,000 획득
- **2009년**: Paul Graham (Y Combinator) 투자; 회사 간신히 생존
- **2011-2015**: 완만하고 꾸준한 성장 (모든 도시에서 규제 싸움)
- **2016년**: $68/주로 IPO; $300억+ 가치
- **2020년 (COVID)**: 국내 여행 폭발; 사람들이 장기 숙박을 위해 집 렌트
- **2023년**: 시가총액 약 $1000억+

**불황 시에도 성공한 이유:**
1. **불황 타이밍 우위** (역직관적):
   - 주택 소유자는 빈 방을 가짐; 수입 필요 (위기로 인한 공급 창출)
   - 여행자는 싼 숙박을 원함 (위기로 인한 수요 창출)
   - 신용 경색은 새 호텔 건설을 의미하지 않음 (경쟁 감소)
   - 노동력은 싼 비용 (사람들이 긱 일자리를 필사적으로 원함)

2. **S자 곡선 변곡점들**:
   - **2008-2010**: 공유경제 개념 신흥; 신뢰 여전히 낮음; 기술 플랫폼 비용 높음
   - **2011-2015**: 스마트폰 침투가 임계량에 도달; 사람들이 낯선 사람 예약에 편함; 결제 시스템 신뢰됨
   - **2016+**: 공유경제 정상화; 규제 수용; 네트워크 효과 지배적

3. **기술 준비**:
   - GPS, 사진 업로드, 디지털 결제가 있는 스마트폰 (2010년 후)이 플랫폼을 가능하게 함
   - 클라우드 인프라가 단위 경제를 낮춤
   - 리뷰/신뢰 시스템이 증명됨 (eBay 모델, Amazon)

4. **네트워크 효과**:
   - 각 호스트가 손님을 끌어당김; 각 손님이 호스트를 끌어당김
   - 임계량에 도달하는 데 8+ 년 소요, 그 후 지수

**핵심 수치:**
- 2008-2011: 창업자는 라면 살이; 겨우 1,000개 등재
- 2015: 100만 등재; $250억 가치 (Series D 후)
- 2020년 (COVID 붐): 예약한 밤 25%+ YoY 증가
- 2023년: 700만+ 등재; 1억 5000만+ 연간 도착

**교훈**: Airbnb의 "너무 일찍" 진단이 틀렸다. 불황에 완벽한 타이밍 (역순환 우위)이며 스마트폰 도입곡선에 맞다.

### 패턴 3: 불황의 역설 — 위기는 창업자의 우위

#### 작동조건
**Fortune 500의 절반은 위기나 불황 중에 설립되었다.**

역직관적으로, 침체는 창업자 우위를 만든다:
1. **노동력이 싼 비용**: 절망이 깊은 재능 풀을 낮은 비용으로 만들 수 있음
2. **자산이 싼 비용**: 부동산, 장비, 중고 인프라는 생존 가격으로 구매 가능
3. **자본 (생존자들을 위해)은 귀중**: 심각한 투자자들로부터의 벤처 자본에 대한 경쟁 감소
4. **소비자 행동 변화**: 효율성, 저가, 새로운 비즈니스 모델에 대한 수요
5. **기존자 산만**: 기존 회사는 생존에 집중, 혁신 아님

#### 인과메커니즘
불황 환경은 강제로 자본 효율성을 높이고, 약한 경쟁자를 빠르게 제거하며, 생존자가 회복기에 규모를 조성하기 위해 남겨진 공백을 활용할 수 있게 한다.

#### 경계조건
- 전체 침체를 견뎌야 함 (자본, 현금흐름, 팀 유지)
- 생존할 수 있는 단위 경제 필요

#### 사례: Google (1998 설립 → 1999 부트스트랩 → 2000 버블 붕괴 → 2004 IPO $850억)

**타임라인:**
- **1998년**: Larry Page와 Sergey Brin이 Google 설립; Stanford 자원으로 부트스트랩
- **1998-1999**: 검색 엔진 전쟁은 혼잡 (AltaVista, Yahoo, Infoseek, Lycos)
- **2000년 3월**: Nasdaq이 5,048.62에서 피크 (1995년 이후 600% 이득); 닷컴 버블이 피크에 도달
- **2000년 3월 - 2002년 10월**: Nasdaq은 78% 붕괴; 수백 개의 인터넷 회사 실패
- **2000-2002**: 붕괴 중, Google은 AdWords를 발명 (2000)하고 광고 경매 모델을 완성
- **2001-2003**: Google은 경쟁자들이 파산하는 동안 수익 3배 성장
- **AltaVista**: Yahoo에 의해 파편으로 구입 후 2003년 폐쇄
- **Infoseek, Lycos, Excite**: 2003년까지 기본적으로 멸종
- **2004년**: Google은 $85/주로 IPO; $230억 가치

**Google이 붕괴를 이긴 이유:**
1. **경쟁자 제거**:
   - Webvan, Pets.com, Boo.com, eToys, iQVC: 모두 2002년까지 죽음
   - 복잡한 검색 엔진 (AltaVista)은 비싼 엔지니어 필요; 급여 삭감을 견딜 수 없음
   - Google의 단순하고 효율적인 설계는 더 적은 리소스 필요

2. **자본 우위**:
   - Google은 강한 기업 파트너십 (Yahoo, AOL)이 붕괴 중 수익 제공
   - 벤처 투자자는 어떤 수익성 있는 회사를 찾으려 필사적; Google의 AdWords가 그것
   - 경쟁자들은 수익성 모델 없이 벤처 자본을 태움

3. **제품 우위**:
   - Google의 PageRank 알고리즘은 우월; 단순한 UI는 더 빠른 페이지 로드를 의미
   - AdWords 경매 모델 (2000)은 천재: 광고주가 입찰하게 하고, 자동화된 효율
   - 경쟁자들은 부풀려진 광고 모델을 가짐; 불황 중 지속 불가능

4. **변곡점의 타이밍**:
   - **1995-2000**: 버블 단계; 비이성적 경쟁; 많은 검색 엔진 자금 조성됨
   - **2000년**: 과장 피크; Google은 여전히 알려지지 않음 (Yahoo, AltaVista보다 작음)
   - **2000-2002**: 붕괴 단계; 비이성적인 것들 죽음; Google의 효율성이 우월임을 증명
   - **2002-2004**: 회복 단계; Google이 65%+ 검색 시장 점유율 캡처

**핵심 수치:**
- 수익: $8600만 (2001) → $4억 3900만 (2003) → $32억 (2005)
- 경쟁자 수: 1999년 15+ 주요 검색 엔진 → 2004년 3개 생존 가능 → 2008년까지 1개 승자

**교훈**: Google은 검색을 발명하지 않았다; 경쟁자들이 죽으면서 확장했다.

#### 대조사례: Pets.com (1998 설립 → 2000년 2월 IPO → 268일 후 2000년 11월 파산)

**타임라인:**
- **1998년**: Pets.com 설립; 벤처 자금 조성 (후기 버블 단계)
- **2000년 2월**: 약 $7억 5000만 가치로 IPO
  - **치명적 타이밍**: Nasdaq 피크의 1개월 전 (2000년 3월 10일)
  - **비즈니스 모델**: 수익성 없음; 누적 손실이 수익을 초과
- **2000년 3월**: Nasdaq이 피크; 닷컴 붕괴 즉시 시작
- **2000년 8월**: Pets.com의 벤처 자금 고갈; 추가 자금 가능 없음
- **2000년 11월**: 파산 신청; IPO 268일 후 청산

**Airbnb와 Google이 생존한 이유와 달리 실패한 이유:**
1. **지속 불가능한 단위 경제**:
   - 펫 음식은 낮은 마진 상품; 배송 비용이 제품 마진보다 많음
   - 고객 획득 비용 >> 생애 가치
   - 10배 규모에서도 수익성으로의 길이 없음

2. **자본 절벽의 타이밍**:
   - IPO는 *붕괴 직전*에; 형편없는 운
   - Amazon은 3월 2000년 붕괴 1개월 전에 최종 대형 라운드 조성; 생존
   - Pets.com은 활주로 없음; $100백만+ 9개월에 타서 수익 없음

3. **불황은 단위 경제에 자비를 베풀지 않음**:
   - 불황은 Airbnb를 도움 (공급/수요 변화)
   - 불황은 Pets.com을 죽임 (배송이 싸도; 비즈니스 모델 깨짐)
   - 차이: Airbnb의 모델은 어떤 경제 규모에서든 작동; Pets.com은 어떤 규모에서도 작동하지 않음

4. **사용 불가능한 인프라**:
   - 창업자 Julie Wainwright (2021 인터뷰): "플러그 앤 플레이 로지스틱 솔루션이 없었다"
   - 2000: 3PL 네트워크가 없음; 모든 것을 커스텀으로 구축해야 함
   - 2020년대: Instacart, Amazon Fresh는 인프라가 존재했기 때문에 성공

**교훈**: 호황 중에 설립된다는 것이 불황 승자를 만들지 않는다.

### 패턴 4: 첫 번째 진입자 신화 — 속도가 타이밍을 이긴다

#### 작동조건
**신화**: "첫 번째로 시장에 나가는 것이 항상 이긴다."
**현실**: 첫 번째 진입자의 47%는 실패; 빠른 추종자의 8%는 실패.

**연구** (Golder & Tellis, 1993; Harvard Business School; Wharton):
- **첫 번째 진입자는 47%가 실패**; 평균 시장점유율 약 10% 캡처
- **빠른 추종자는 8%가 실패**; 평균 시장점유율 약 28% 캡처
- **빠른 추종자는 53% 시장에서 승리** vs. 첫 번째 진입자는 37%

#### 인과메커니즘
두 번째 진입자는: (1) 첫 번째 진입자의 실수에서 배움, (2) 기술 개선된 이점, (3) 소비자가 이미 첫 번째 진입자의 마케팅으로 검증됨

#### 경계조건
- 시장이 명백히 준비되어야 함 (첫 번째 진입자가 증명)
- 더 나은 실행이 가능해야 함
- 진입하기 전에 네트워크 효과가 락인되지 않아야 함

#### 사례: Facebook (2004) vs. Friendster (2002)

**타임라인:**
- **2002년 3월**: Friendster 출시; 첫 소셜 네트워크
- **2003-2004**: Friendster가 폭발; 1년 미만에 300만 사용자; 소셜 네트워킹 지배
- **2004년 4월**: MySpace 출시; Friendster 실수에서 배움; 이를 능가
- **2004년 2월**: Facebook (TheFacebook) Harvard에서 출시; 첫 날 1,200 가입
- **2004-2005**: Facebook은 단계별 대학 확장; 2004년 말 100만 사용자
- **2006년**: Facebook이 고등학교를 열고; 13세 이상 모두에게 열고; Instagram 스타일 사진 태깅
- **2008년**: Facebook이 MySpace를 능가; 지배적 소셜 네트워크가 됨
- **2015년**: Friendster 완전 폐쇄

**Facebook이 시장의 두 번째임에도 승리한 이유:**
1. **Friendster의 실행 실패**:
   - **2002-2004**: 폭발적 성장 → 서버 압도 → 페이지 로드 40초
   - **형편한 확장성**: 인프라는 성장을 처리할 수 없음; 사용자 경험 악화
   - **감소하는 참여**: 사용자가 좌절해서 떠남; 이탈 가속
   - **MySpace에 졌음**: MySpace는 Friendster 실수에서 배우고 더 잘 실행
   - **Google 제안 (2003)**: 창업자는 $3000만 거절, 승리한다고 생각; 대신 졌음
   - **지역적 초점**: Friendster는 아시아에서 강함; 미국에서 약함; 글로벌 네트워크 효과 캡처 안 함

2. **Facebook의 통제된 성장 전략** (Friendster의 반대):
   - **2004-2006**: 의도적으로 성장 제한, 대학 네트워크에만
   - **확장성**: 각 확장은 인프라 준비와 매칭됨
   - **제품 속도**: 혁신적 기능 출시 (2006년 News Feed, 사진 태깅, 2008년 모바일)
   - **미국 초점**: 기본 시장 (미국 십대) 캡처 후 국제화; 미국 = 최대 디지털 경제
   - **실행**: 최고 엔지니어 고용; 심각한 투자자로부터 자본 조성 (Accel, Sequoia)

3. **기술 준비 정렬**:
   - **광대역 확장**: 2002-2008은 미국의 대규모 광대역 채택을 봄
   - **사진 공유 규범**: Instagram은 2010까지 없음; Facebook이 사진 태깅을 발명 (2005)
   - **모바일 불가피**: 스마트폰 곧 도착 (iPhone 2007); Facebook은 초기에 모바일 우선 문화 준비

4. **S자 곡선 타이밍**:
   - Friendster는 과장 단계 진입 (초기 2000년대 인터넷 흥분)
   - Facebook은 변곡점 단계 진입 (2004-2008, 광대역 유비쿼터스, 사회 규범 변화)

**핵심 수치:**:
- Friendster: 300만 (2003) → 1억 1500만 (2008) → 폐쇄 (2015)
- Facebook: 100만 (2004) → 10억 (2012) → 30억 (2024)
- MySpace (두 번째): 7500만 (2008) → 파산 (2011)

**교훈**: MySpace와 Facebook 모두 Friendster를 이겼다; 더 빨랐기 때문이 아니라 더 잘 실행했기 때문.

### 패턴 5: 거시 타이밍 — 자본 순환과 경제 윈도우

#### 작동조건
**자본 가용성은 하드 제약이다.** 스타트업은 제품-시장 부합뿐 아니라 신용 순환의 올바른 지점에서 자본을 조성해야 한다.

- **호황 단계** (2005-2007, 2013-2021): 쉬운 돈; 더 많은 경쟁; 낮은 배수
- **침체 단계** (2000-2002, 2008-2009, 2020 3월 락다운): 돈이 빠듯함; 생존 회사는 활주로 있음; 경쟁 덜함
- **회복 단계** (2003-2004, 2010-2012, 2021+): 성장에 이상적; 승자 명확; 승자를 위한 자본 복귀

#### 인과메커니즘
경제 순환의 단계는 자본 가용성, 경쟁 강도, 모범 사례에 대한 투자자의 관심을 결정한다. 회복 단계에서 증명된 모델은 신속한 자금 조성을 볼 수 있다.

#### 경계조건
- 강한 단위 경제 필요 (호황에서 생존하려면)
- 자본 효율성 필요 (침체에서 생존하려면)

#### 사례: Uber (2009 설립 → 완벽한 타이밍 수렴)

**타임라인:**
- **2009년 3월**: Garrett Camp와 Travis Kalanick이 Uber를 설립
  - **맥락**: 글로벌 금융 위기 피크; 일자리 사라짐; 투자자 회의적
- **자금 조성 겨울 (2009-2010)**: Series A 조성에 12개월 이상 걸림; MVP를 부트스트랩해야 함
- **2010년 5월**: San Francisco에서 작동 MVP로 출시 (겨우; "버그 많고 겨우 작동")
- **2010-2014**: 폭발적 성장; 회복이 가속되면서 벤처 자본 캡처
- **2015년**: $15억을 $500억+ 가치로 조성; 유니콘이 됨

**타이밍이 완벽했던 이유 (거시 × 기술 × 행동)**:
1. **경제 타이밍**:
   - **2009년**: 불황은 택시 운전자들이 일을 필사적으로 원함; 낮은 모집 장벽
   - **2009년**: 조성된 자본 감소: Slack 자본이 의미 Uber는 모델을 린으로 증명해야 함 (효율적)
   - **2010-2014**: 회복 가속; 벤처 자본 복귀; Uber는 "이야기" + 견인력을 가짐

2. **기술 성숙**:
   - **GPS**: 위성 포지셔닝 성숙; 통신사 (Verizon, AT&T)가 인프라 구축
   - **스마트폰**: iPhone (2007) 및 Android (2008-2010) 침투가 임계량에 도달
   - **결제 시스템**: 신용카드 처리 상품화됨; Square, Stripe는 2009-2010 출시
   - **지도**: Google Maps API 사용 가능; 자신의 구축 필요 없음

3. **행동 변화**:
   - **택시 불신**: 택시 메달리온 시스템은 부패로 인식됨; 소비자는 투명성 원함
   - **긱경제 정상화**: 2010년대는 부분시간, 유연한 일의 정상화를 봤음
   - **낯선 사람 신뢰**: Airbnb (2008) 및 초기 Uber 리뷰가 낯선 사람 거래를 정상화
   - **스마트폰을 지갑으로**: 비접촉 결제가 규범이 되고 있음

4. **규제 공백**:
   - **2009-2014**: 도시들은 아직 라이드셰어를 규제하지 않음; 첫 진입자 우위는 거대
   - **2015년 이후**: 규제가 Uber를 기존자로 락인; 새 진입자는 법적 장벽에 직면

**핵심 수치:**
- 설립에서 $10억 가치: 약 6년 (2009 → 2015)
- 2020년 GMV: 글로벌 $910억
- 피크 팬데믹 영향: 탑승 70% 감소, 배달 300% 증가; 생존

**교훈**: Uber는 네 가지 타이밍 요소가 정렬되었기 때문에 성공:
1. 경제 위기가 노동 공급을 창출
2. 기술은 성숙 (출혈 우위, 증명됨)
3. 소비자 행동 변화 (불황 후 인색함 + 스마트폰 정상화)
4. 자본이 규모 전에 경쟁자들로 복귀

어떤 한 가지 요소가 실패하면 Uber는 존재하지 않는다.

---

### 패턴 6: S자 곡선 읽기 — 과장 주기 vs. 채택 현실

#### 작동조건
**기술은 과장 주기와 맞지 않는 예측 가능한 채택곡선을 통과**:

1. **과장 피크** (5-10년 채택 전): 미디어 흥분; 벤처 자금; 많은 경쟁 제품
2. **실망의 골짜기** (과장 후 0-3년): 제품 실패; 초기 채택자 포기; 자금 고갈
3. **계몽의 슬로프** (3-10년): 헌신적 실무자들이 실질적 사용 증명; 단위 경제 개선; 채택 가속
4. **생산성의 고원** (10+ 년): 유비쿼터스; 상품화; 네트워크 효과 설정

#### 인과메커니즘
과장이 사라진 후, 헌신적 실무자들이 실질적 가치를 증명하기 시작할 때 가장 가파른 성장이 발생한다. 이것은 낮은 기대와 높은 실제 효용의 시점이다.

#### 경계조건
- 과장 피크에서 진입하면 보장된 실망
- 골짜기에서 진입하면 자본이 있으면 우위, 없으면 사망
- 슬로프에서 진입하면 성장 가속화를 캡처

#### 사례: Slack의 "느린" 성장 단계 (2013-2016)

**타임라인:**
- **2013년 8월**: Slack 베타, 첫 주 15,000팀 출시
- **2013-2014**: 매체 수준 과장; 초기 채택자 가입; 사용 열정적
- **2014-2016**: 성장은 꾸준하지만 지수적이 아님; 2016년까지 250,000팀; $27억 가치
  - 이 기간은 종종 "느림" 또는 "고원"으로 무시됨
- **2016-2019**: 성장이 가속; 채택이 주류가 됨; 2019년 $195억으로 IPO
- **S자 곡선 현실**: 2013-2016은 "계몽의 슬로프" — 백분율 측면에서 곡선이 가장 가파를 때

**"느린" 성장이 실제로 최적 타이밍이었던 이유:**
- **2013-2014**: 과장 단계; 많은 경쟁자 출시 (HipChat, Slack, Microsoft는 결국 Teams 구축)
- **2014-2016**: Slack은 제품이 진정으로 더 좋았기 때문에 생존; 네트워크 효과 축적; 전환 비용 구축
- **2016-2019**: 경쟁자들 (Teams, HipChat)은 성장 단계 중에 도착; 너무 늦음; Slack이 이미 승리
- **교훈**: 과장 중에 느린 성장은 종종 강점 — 당신은 다른 것들이 죽는 동안 계속 구축하고 있다.

---

#### 대조사례: Google Glass — 과장 피크에서 출시

**타임라인:**
- **2012년**: 발표; 과장 피크; Time Magazine "올해의 최고 발명"
- **2013-2014**: 과장 고원; 초기 리뷰 섞임; 제품은 실제 제한이 있음
- **2014-2015**: 과장 붕괴; 제품 중단됨
- **2023+**: AR/VR 채택곡선이 변곡점에 도달; 성공 경쟁자들 (Meta Quest, Apple Vision Pro)

**Glass가 과장 피크에서 실패한 이유:**
1. **과장 후, 골짜기가 아닌 피크에서 출시**:
   - 과장 피크 = 가장 높은 기대 = 보장된 실망
   - 제품은 "베타" 품질; 마법을 기대하는 소비자를 위해 준비 안 됨

2. **실망의 골짜기 곧 도착**:
   - 2013-2015: AR 과장 주기 붕괴; 투자 고갈
   - Glass는 제품이었기 때문에 골짜기를 견딜 수 없음; 플랫폼이 아님

3. **경쟁자들은 골짜기 후에 도착**:
   - 2023: Meta Quest 3, Apple Vision Pro는 골짜기 후에 출시
   - 2023년까지, 소비자 기대는 정상화됨; AR은 증명됨 가치 (게임, 엔터프라이즈)
   - 배터리 수명, 무게, 가격 모두 해결됨

**교훈**: 과장 피크에서 출시하지 마라. 골짜기 후, 기대가 낮고 현실적일 때 출시해라.

---

### 패턴 7: 경쟁 타이밍 — 실행이 타이밍을 이길 때 두 번째 진입자 우위

#### 작동조건
시장 조건이 올바르지만 실행이 다를 때, 두 번째 진입자는 다음 이유로 종종 이긴다:
1. 첫 진입자는 자본 부담 (탐색에는 비싸고; 낭비적)
2. 두 번째 진입자는 첫 진입자의 실수에서 배움 (1-2년의 무료 데이터)
3. 시장 진입 사이의 기술 개선 (해자 약화)
4. 첫 진입자의 마케팅 지출로 검증되는 소비자 행동

#### 인과메커니즘
학습, 기술 개선, 검증된 시장은 두 번째 진입자에게 낮은 초기 실행 리스크로 더 빠른 스케일링을 가능하게 한다.

#### 경계조건
- 첫 진입자가 5+ 년 이상 자본을 견딜 수 있으면 보통 이기는 경향
- 시장이 명백히 준비되면, 빠른 추종자는 종종 이김 (Palm vs. Newton)

#### 사례: Palm (1997) vs. Apple Newton (1993)

**타임라인:**
- **1993년**: Apple Newton 출시; $700 가격표; 필기 인식 실패; 미디어에 의해 조롱됨
- **1993-1997**: Newton이 투쟁; 높은 비용, 형편한 필기, 느림
- **1997년**: Palm Pilot 출시; $300 가격표; 단순하고 효과적인 Graffiti 필기 시스템
- **1997년**: Palm이 몇 달 내에 PDA 시장의 66% 캡처
- **1998년**: Palm Pilot은 기본 PDA가 됨; Newton 중단됨

**Palm이 4년 늦음에도 승리한 이유:**
1. **Newton에서 배움:**
   - Newton은 너무 야심적 (AI 필기); 실패
   - Palm은 달성 가능한 기능 세트 (기본 Graffiti)로 단순화
   - Newton의 $700 가격이 시장을 가르침: PDA는 <$400여야 함

2. **기술 성숙**:
   - 1993-1997: 프로세서 개선; 비용 감소
   - Palm은 50% 가격에 더 나은 성능을 제공할 수 있음
   - Newton 팀이 R&D에 수백만을 태움; Palm은 그 일에서 이득

3. **실행 차이**:
   - Newton: Apple 하드웨어 엔지니어로 구축; 복잡
   - Palm: 휴대 기기에서 온 사람들로 구축; 본질에 집중
   - Palm은 Newton이 무시한 파트너십 (데스크톱 동기화)에 투자

4. **시장 타이밍**:
   - 1993: PDA 개념은 새로움; 소비자 회의적
   - 1997: PDA는 수용됨; 소비자 기대는 현실적
   - Palm은 "계몽의 슬로프"에 대해 진입, 과장 피크가 아니라

**핵심 수치:**
- Newton: 5년간 약 600,000개 판매; $5억+ 손실
- Palm: 18개월에 100만 개 판매; 첫 날부터 수익

**교훈**: 시장이 두 번째에 두 번째 진입자인 것이 불리가 아니다 (1) 첫 진입자가 고객을 교육했다면, (2) R&D 비용을 흡수했다면, (3) 시장이 존재함을 증명했다면.

## 실패 패턴 라이브러리

### 패턴 F1: 너무 일찍 — 기술과 시장 준비 부족

#### 작동조건
기술이 불안정하고, 인프라가 존재하지 않으며, 소비자가 채택하기를 거부할 때 진입.

#### 인과메커니즘
자본을 소진하고 고객 저항, 기술 실패, 기다리는 시간의 경제학으로 죽는다.

#### 사례: Google Glass, Webvan (위 참조) & Pets.com

#### 경계조건
- 네트워크 효과가 있으면 충분한 자본으로 견딜 수 있음 (Airbnb)
- 단위 경제가 깨지면 아무리 자본도 저장할 수 없음 (Pets.com, Webvan)

---

### 패턴 F2: 너무 늦게 — 네트워크 효과와 기존자 우위

#### 작동조건
시장이 성숙하고, 기존자가 네트워크 효과를 축적했으며, 진입 장벽 (경제, 규제, 기술)이 높을 때 진입.

#### 인과메커니즘
신규 진입자는 비용 불리, 낮은 사용자 채택, 승자가 정해진 시장의 가격 압박에 직면한다.

---

### 패턴 F3: 실행 실패 — 기술 준비는 맞지만 제품이 깨짐

#### 작동조건
S자 곡선 타이밍은 올바르지만 제품이 약하거나, 대규모로 확장하지 않거나, 경쟁자가 더 잘 실행한다.

#### 인과메커니즘
첫 진입자는 실수의 비용을 모두 부담하고, 빠른 추종자가 더 낮은 리스크로 시장을 캡처한다.

#### 사례: Friendster, Newton (위 참조)

## 진단 체크리스트

### 지금 진입해야 하는가?

**S자 곡선 진단:**
- [ ] 고객이 "준비됨"이라고 말하는가? (추측하지 말고 물어보기)
  - "아니, 하지만 2-3년에 준비될 수도" → 너무 일찍. 기다리거나 피벗.
  - "네, 하지만 가격이 잘못됨" → 변곡점. 진행.
  - "아마도" → 채택곡선 보기; 아직 진입하지 말 것.

**자본 순환 진단:**
- [ ] 우리는 어느 단계에 있는가?
  - 호황 (2021-현재?): 서 있기 어려움; 더 많은 경쟁; 낮은 배수
  - 침체 (2022-2024?): 자본 빡빡; 강한 경쟁자 수요; 수익 경로 증명 필요
  - 회복: 이상적. VCs는 승자를 원함.

**단위 경제 진단:**
- [ ] 3-5년 손실을 견딜 수 있는가?
  - Airbnb: 예 (보육 모금 + 라면 생존)
  - Webvan: 아니오 (수익성으로의 경로 없음)
  - Zoom: 예 (고객이 지불함; 성장 100% YoY)

**경쟁 타이밍 진단:**
- [ ] 시장이 증명되었지만 여전히 지배적이지 않은가?
  - 증명됨 + 지배적 (2019년 스트리밍 후기): 어려움. 3-4 플레이어.
  - 증명됨 + 신흥 (2004년 소셜 네트워크): 좋음. 많은 승자 가능.
  - 증명 안 됨 (2012년 모바일 지갑): 위험. 존재하지 않을 수 있음.

### 진입 판정

**진행하라** if:
- [ ] S자 곡선이 가파름 (20-60% 채택)
- [ ] 단위 경제는 지속 가능
- [ ] 자본 순환 허락 (호황 또는 회복 단계)
- [ ] 2-3개 주요 경쟁자 존재하지만 시장 포화 아님

**기다려라** if:
- [ ] S자 곡선이 아직 얕음 (<20% 채택)
- [ ] 단위 경제는 보조금 의존적
- [ ] 자본 순환이 침체 (부트스트랩할 수 없으면)
- [ ] 0 경쟁자 (시장 검증 안 됨) 또는 10+ 경쟁자 (시장 포화)

**피벗하거나 다시 시작하라** if:
- [ ] 타이밍은 틀렸지만 아이디어는 올바름 (2-3년 후 회수)

## 조건부 전략 매트릭스

### Axis 1: 시장 준비 (증명됨 vs. 미증명)

| 증명됨 | 미증명 |
|--------|--------|
| **전략: 빠른 실행 + 네트워크 효과** | **전략: 천천히 구축 + 신뢰** |
| 예: 2015년 이후 Slack, Uber, Airbnb의 그룹 | 예: 2008년 Airbnb, 2011년 Zoom |
| 고객이 원하는 것을 알고 있음 | 고객이 원하는 것을 모름 |
| 빠른 실행이 이기는 경향 | 제품 완성도가 이기는 경향 |
| 목표: 네트워크 효과 먼저 | 목표: 단위 경제 증명 |

### Axis 2: 자본 가용성 (충분 vs. 제한됨)

| 충분 | 제한됨 |
|------|--------|
| **전략: 공격적 확장** | **전략: 효율성 우선 + 수익** |
| 예: 2015-2021년 Uber, SoftBank 자금 | 예: 2008-2010년 Airbnb |
| 시장 점유율 우선 | 장기 생존 우선 |
| 목표: 규모로 경쟁자 능가 | 목표: 수익성의 경로 증명 |
| 위험: 낭비적 지출, 번 속도 | 위험: 현금 부족, 느린 성장 |

### Axis 3: 경쟁 강도 (낮음 vs. 높음)

| 낮음 | 높음 |
|------|------|
| **전략: 시장 교육 + 기준 설정** | **전략: 차별화 + 실행 우위** |
| 예: 2006년 Shopify, 2013년 Slack (초기) | 예: 2004년 Facebook vs. Friendster |
| 당신의 표준을 정의할 시간 있음 | 기준은 이미 설정됨 |
| 목표: 시장 지배력 | 목표: 경쟁자 능가 |
| 위험: 시장 검증 지연 | 위험: 가격 압박, 기능 경쟁 |

### Axis 4: 기술 준비 (성숙 vs. 초기)

| 성숙 | 초기 |
|------|------|
| **전략: 빠른 확장** | **전략: R&D 투자 + 인내** |
| 예: 2007년 Netflix, 2019년 Zoom IPO | 예: 2012년 Google Glass, 2008년 Webvan |
| 기술 리스크 낮음 | 기술 리스크 높음 |
| 목표: 시장 캡처 | 목표: 기술 증명 |
| 위험: 경쟁자도 빠르게 움직임 | 위험: 자본 소진 (증명 전) |

## 축간 연결

| 축 | 연결 패턴 | 예시 |
|----|----------|------|
| **F1 시장 (문제-솔루션 부합)** | 시장이 문제가 명확할 때만 타이밍 최적. Slack은 이메일 통증이 명백했기 때문에 성공; Glass는 문제가 불명확 (왜 필요?). | Slack (명백한 문제) vs. Glass (불명확한 필요성) |
| **G1 0→1 (아이디어 독창성)** | 창의적 아이디어는 타이밍과 독립적으로 너무 일찍 올 수 있음. Google Glass는 창의적이지만 시기 부적절; Netflix는 창의적이면서도 시기 적절했음 (기술이 따라잡음). | 창의성 ≠ 타이밍. 둘 다 필요. |
| **S1 피벗 (재전략화)** | 타이밍이 틀렸으면 2-3년을 기다렸다가 재입장하거나 제품/시장 피벗. Airbnb는 2008년에 타이밍이 맞아떨어졌지만 그렇지 않았다면 다시 시작해야 함. | 타이밍 재진단 시 재시작 전 새로운 윈도우 기다리기. |
| **E2 자본 (자금 조달)** | 자본 순환과 타이밍은 동의어. IPO는 시장 가치와 자본 순환 모두에 따라 달라짐. Uber는 회복 단계에서 자금 조성했기 때문에 성공; Pets.com은 호황 피크에서 IPO했기 때문에 실패. | IPO 타이밍 = 거시 + 기업 성숙도. |
| **E3 위기 (생존과 회복)** | 위기 중에 설립된 회사는 보통 회복기에 번성. Google, Airbnb는 불황 중 설립되었으나 회복기에 규모화 (생존 후). | 위기 설립 + 회복 규모화 = 종종 큰 승자. |

## 결론: 타이밍은 학습 가능

데이터는 **타이밍이 운이 아니라 기술**임을 압도적으로 보여준다. 다섯 가지 능력의 조합:

1. **거시 인식** (비즈니스 순환, 자본 흐름, 규제 변화)
2. **기술 유창성** (S자 곡선, 채택 변곡점, 인프라 준비)
3. **소비자 심리** (행동 변화 표시, 지불 의사 임계값)
4. **경쟁 정보** (시장 성숙, 첫 진입자 vs. 빠른 추종자 역학)
5. **무자비한 자본 규율** (기다리는 기간을 견딜 수 있는가?)

이 패턴들을 연구하고 읽기를 연습하는 창업자들은 "운"으로 보이지만 실제로는 패턴 인식이다.

**최고의 창업자들은 코딩만큼 거시 추세, 채택곡선, 자본 순환을 읽기에 시간을 쓴다.**
