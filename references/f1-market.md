# 시장·고객 선택 — 누구의 어떤 문제를 푸는가?

## 성공 패턴 라이브러리

### P1: 기존 산업 내 구체적 Pain Point 해결 → Viral Growth
**작동 조건**
- 고객이 이미 인식하고 있는 명확한 불편함 존재
- 그 문제를 푸는 방식이 지금까지 없었거나, 매우 비효율적이었음
- 솔루션이 사용할수록 유용하고, 말할수록 설명하기 쉬움

**인과 메커니즘**
- Pain이 크면 클수록 → 고객이 자연스럽게 word-of-mouth 시작
- 제품의 "쾌감"(delight)이 분명하면 → 스스로 다른 사람에게 추천
- 마케팅 없이 viral이 가능 (고객이 마케팅)

**경계 조건**
- Pain point가 '인지적' 필요일 때만 작동 (희망이나 욕망이 아님)
- 제품 자체가 "삶을 바꾼다"는 느낌이 있어야 함
- 타겟 고객층에서 네트워크 효과가 강해야 함

**사례**
- **Slack** (2014): 팀 커뮤니케이션의 분산 문제 해결
  - 기존: 이메일 + 여러 협업툴 (Campfire, HipChat 등)
  - 변화: 15k DAU (Feb 2014) → 1.1M (Jun 2015) → 4M (Oct 2016)
  - 성공 요인: 원격팀의 명확한 pain, 유저 중심 설계, Stewart Butterfield의 직접 Twitter 고객지원
  - 마케팅 예산 거의 0

**대조 사례 (같은 조건에서 실패)**
- **Snapchat** (2011~): 사진 공유의 "임시성" pain을 해결했으나
  - 성공: Gen Z (13-25) 한정
  - 실패: 25세 이상 세대는 UX 진입장벽 높음 (아이콘 설명 없음, 레이블 부재)
  - 이유: Pain이 특정 세대에만 있었고, 다른 세대와의 네트워크 효과 약함

### P2: 니치에서 시작 → 각 카테고리마다 같은 가치제안으로 확장
**작동 조건**
- 단일 카테고리에서 압도적 고객만족도 달성
- 고객만족도 지표(고객 발언, 재구매율)로 증명 가능
- 다른 카테고리도 같은 "불편함의 구조"를 가지고 있어야 함 (선택, 가격, 편의성 등)
- 각 카테고리별로 충분한 규모 필요

**인과 메커니즘**
- 한 범주에서 신뢰 확보 → 같은 로직을 다른 범주에도 적용
- 물류·유통 인프라 재사용 가능 → 한계비용 감소
- 고객 데이터(구매 패턴, 세그먼트)를 다음 범주에 활용

**경계 조건**
- 첫 번째 카테고리에서 실제 PMF 달성이 前提
- 확장 속도가 너무 빠르면 brand promise 훼손 가능
- 각 카테고리의 경쟁 구도가 다르면 전략 재조정 필요

**사례**
- **Amazon** (1994~): 책 → 음악 → 전자제품 → 모든 것
  - 1994-1997: 책의 선택/가격/편의성 문제 해결 (earth's biggest bookstore)
  - 1997 IPO로 $54M 자금 조달
  - 후속 확장: 음악(late 90s) → DVD → 전자제품 → 장난감
  - 원칙: 각 카테고리마다 같은 가치제안 (selection, price, convenience)
  - 결과: 2025년 시총 $2.36조, $5.9B revenue (2004) 성과 달성

**대조 사례 (니치에 머물거나 잘못 확장)**
- **Fitbit** (2009~): 피트니스 트래킹 니치에서 early mover advantage
  - 성공: 2009 피오니어 → 2016년 $2B 수익, 30% 시장점유율
  - 실패: Apple Watch, Xiaomi, Huawei와의 확장 경쟁에서 후퇴
  - 2021년 Google에 $2.1B에 인수 (극적 가치 손상)
  - 이유: 니치에서의 승리가 다른 카테고리(스포츠워치, 수면추적)로 확장하는 데 충분하지 않았음

### P3: 산업 재정의 — 기존 틀을 깨는 수요 창출
**작동 조건**
- 기존 산업의 고정관념(가격, 접근성, 품질의 삼각형)을 도전
- 고객이 원래 "포기했던 수요"를 발굴
- 새 수요층이 존재하지만 공급자가 없었던 상황

**인과 메커니즘**
- 기존 시장 점유자의 "고정비용 + 기득권"이 가격/접근성 차단
- 신입자가 그 제약을 우회하면 → 새로운 시장층 오픈
- 초기 신규 수요자 확보 → 규모의 경제 → 기존 산업 비교 우위

**경계 조건**
- 진정한 재정의는 가격 or 접근성 one of 요소로는 불충분 (기존 경쟁사도 따라할 수 있음)
- 재정의의 차원이 "문화적" or "패러다임적" 수준이어야 함

**사례**
- **Airbnb** (2008~): "호텔" 산업 재정의 → 개인 숙박 → 경험
  - 기존 구조: 호텔체인(높은 고정비, 일관된 서비스, 표준화)
  - Airbnb 변화: 개인 호스트 → 인증 시스템 → 고유 경험
  - 수요 발굴: 저가 숙박 + 현지 경험을 원하는 여행객
  - 공급 발굴: 유휴 공간 활용하려는 집주인
  - 성장: 4M listings, 2020년 150M 사용자
  - 호텔 산업 영향: 직접 수익 손실 ~$450M/year

**대조 사례 (재정의 시도 but 실패)**
- **Juicero** (2013~2017): Cold-press 주스 기계
  - 시도: "자동화된 주스"를 프리미엄 경험으로 재정의
  - 실제 문제: 수요가 없었음 (손으로 짜도 되는데 $699에 기계?)
  - 기술적으로는 정교했음 (400개 custom parts) ⟹ 제조 어려움 + 가격 거품
  - 가격 인하 시도: $699 → $399 (Jan 2017) ⟹ 여전히 부족
  - 적자: $4M/month
  - 종료: 2017년 9월, 16개월 만에 서비스 중단

### P4: 개인화·큐레이션으로 "정보 과부하" 문제 해결
**작동 조건**
- 카테고리 내 선택지가 너무 많아서 고객이 결정 마비(decision paralysis) 상태
- 기존 필터(추천, 랭킹)가 충분하지 않음
- 고객 데이터(행동, 선호)를 충분히 수집 가능한 구조

**인과 메커니즘**
- 개인화 알고리즘 → 고객 선호도 확률 증가 → 사용 빈도 증가
- 자신만의 "발견" 경험 → brand loyalty 강화
- 플랫폼이 더 자주 쓰일수록 → 더 정교한 데이터 → 더 나은 추천

**경계 조건**
- 초기 "cold start" 문제: 데이터 없을 때 품질 낮음
- 개인화가 "필터 버블" 되면 고객 이탈 가능 (다양성 포기)

**사례**
- **Spotify** (2008~): 음악 스트리밍의 "무한 선택" 문제 해결
  - 기존: iTunes(구매), Pandora(라디오) → 선택 어려움
  - Spotify 변화: Freemium 모델 + 개인화 추천 (2015년 "Discover Weekly" 출시)
  - 큐레이션: 매주 맞춤 플레이리스트 → 자연스러운 발견
  - 바이럴: Spotify Wrapped (음악 취향 공유) → 소셜 미디어 유기 확산
  - 성장: 2008년 유럽 10M → 2011년 미국 진출 → 2025년 299M, 35% 글로벌 시장점유

### P5: 지역 로컬라이제이션 + 문화 맞춤 — 글로벌 플랫폼의 로컬 착근
**작동 조건**
- 글로벌 플랫폼이 로컬 문화·행동 패턴 깊게 이해
- 각 지역별 콘텐츠/기능을 현지화 (단순 언어 번역 아님)
- 지역 네트워크 효과를 강화하는 기능 (예: 이모지, 캐릭터, 지역 콘텐츠)

**인과 메커니즘**
- 로컬 맞춤 → 고객이 "이 앱은 우리 문화를 이해한다"고 느낌
- 높은 engagement → 초기 시장 장악 → 후발주자 추격 어려움

**경계 조건**
- "과도한 로컬라이제이션"은 글로벌 스케일 저해 가능
- 로컬 맞춤이 타 지역에서 외면당할 수 있음

**사례**
- **LINE** (2011~): 일본/대만 시장 지배
  - 출발: 2011년 일본 지진/쓰나미 이후 disaster recovery 수요 → 매우 빠른 신뢰 획득
  - 로컬라이제이션: 일본식 귀여운 스티커(Brown, Cony 등) → 문화적 공감
  - 지역별 콘텐츠: LINE TV (지역 드라마, K-pop, J-drama)
  - 성과: 178M MAU, 일본 92M (성인 88%), 대만 17M, 태국 33M

### P6: 타이밍 + 기술 준비도 조화
**작동 조건**
- 기술이 충분히 성숙 (사용 불가능할 정도로 덜 익지 않아야 함)
- 소비자가 문제를 "인지할 준비" 됨
- 인프라(인터넷, 디바이스, 결제 등)가 갖춰짐

**인과 메커니즘**
- 기술 준비 + 시장 준비 = timing unlock
- 너무 일찍 출시 → 기술 불안정 + 초기 user 탈락 → brand damage
- 너무 늦음 → 경쟁사 선점 + 시장 포화

**경계 조건**
- "타이밍이 맞다"는 사후적 판단이 쉽지만, 사전 예측은 어려움
- 기술 발전 속도를 과다 추정하는 경향 (Hype Cycle)

**사례**
- **Netflix** (1997~2007): DVD 렌탈 → 스트리밍 전환 성공
  - 1997: Netflix 창립, DVD 렌탈 시작
  - 2000: Reed Hastings 예견: "디지털 미래가 온다" → Netflix를 $50M에 Blockbuster에 제안 (거절당함)
  - 2007: "Watch Now" 스트리밍 출시 (초기 1000개 타이틀)
  - 조건 맞춤: broadband 인프라 + 디바이스(Roku, Xbox, PS) 준비 완료
  - 하이브리드 모델: 스트리밍을 DVD 구독자 무료 추가 기능으로 제공
  - 2010: 2000만 구독자, 스트리밍 > DVD 수익

**대조 사례 (타이밍 실수)**
- **Google Glass** (2013~2015): 시기상조
  - 출시: 2013년 "explorer edition", 2014년 5월 일반 공개
  - 실패: 2015년 소비자 시장 철수
  - 문제: 기술 미성숙, AR/wearable이 덜 익음, 배터리 6-8시간만, 가격 $1500
  - Sergey Brin 회고 (2025): "commercialise too quickly"

### P7: 커뮤니티 → 고객 발견 → 제품 개선의 선순환
**작동 조건**
- 초기 adopter와 깊은 관계 유지 가능한 구조
- 고객이 product feedback을 능동적으로 제공 가능
- product의 customization/personalization 여지가 있음

**인과 메커니즘**
- 작은 초기 커뮤니티 → 제품 개선 데이터 무한 생성
- 개선된 제품 → 커뮤니티 추천 → 확대 고객층
- user-generated content(templates, tutorials) → 진입장벽 낮춤

**경계 조건**
- 커뮤니티 크기가 너무 작으면 feedback 신호 약함
- 커뮤니티 의견과 광시장 수요가 misalign할 수 있음

**사례**
- **Notion** (2013~2018): 5년간의 고객 발견 과정
  - 2013-2017: 4년간 product 출시 안 함 (고객 탐색)
  - 고객 발견: 어떤 고객이 뭘 원하는가? 명확히 할 때까지 시간 투자
  - 약 5년 소요: 고객만족도 확실 → 그 다음 scale
  - 커뮤니티: Template Gallery → user-generated content 축적
  - 마케팅: organic growth + influencer 마크팅
  - 성과: 유저가 템플릿/튜토리얼 직접 창작 → acquisition cost 감소

---

### P8: 개발자 니치 → 플랫폼 지배
**작동 조건**
- 개발자들의 구체적 pain point 해결
- 개발자 커뮤니티 자체가 매우 powerful (word-of-mouth, 자기 회사에 도입)
- 초기 진입장벽 낮음 (free tier, open source 친화적)

**인과 메커니즘**
- 개발자 만족 → 기업 채택 → B2B revenue
- 개발자 community (conference, forum) → 강력한 network effect
- 플랫폼이 점점 essential해짐 (switching cost 증가)

**경계 조건**
- 개발자 시장이 항상 수익성 높지는 않음 (무료 문화)
- 초기 개발자 만족도가 매우 중요 (평판 민감)

**사례**
- **GitHub** (2008~): 개발자 협업 플랫폼 지배
  - 출발: Git version control 중앙화 필요성
  - PMF: 2008-2010년 사이 개발자 커뮤니티 폭발적 성장
  - 특징: Free public repos, social features (follow, star), Issues/PR 워크플로우
  - 성장: 100M+ users (2024), 60% 시장점유율
  - Global reach: 인도 개발자 (20M)가 가장 빠르게 증가 중

## 실패 패턴 라이브러리

### F1: 허상 수요에 속기 — "고객이 원할 것 같은" 것 vs 실제 수요
**작동 조건**
- 제품 담당자가 고객 인터뷰 대신 자신의 직관/가설에 의존
- "이런 게 있으면 좋지 않을까?"의 상위 1% 고객 만족도에 집중
- 실제 구매/사용 의사를 검증하지 않음

**인과 메커니즘**
- Founder vision + VC funding → 제품 제작 → 출시 후 고객이 없음
- Sunk cost fallacy → 가설 검증 대신 marketing 강화 시도 → 더 큰 손실

**경계 조건**
- 가끔 founder vision이 맞기도 함 (Steve Jobs)
- 하지만 most cases에는 early customer validation이 필수

**사례**
- **Juicero** (2013~2017): 허상 수요 + 기술 과신
  - 가설: "바쁜 건강식 애호가들이 fresh juice를 매일 집에서 마시고 싶을 것"
  - 제품: $699 cold-press juicer + subscription juice packs
  - 문제점: 고객 validation 부족, 손으로 짜면 되는데 왜 기계?, 기술 과신 (400개 custom parts)
  - 증거: Bloomberg (2016) ← "손으로 짜도 되잖아?"
  - 결과: 가격 인하 시도 ($699 → $399) → 실패 → 16개월 만에 서비스 중단
  - 손실: $120M 투자 손실

---

### F2: 시장 자체가 없거나 너무 작음
**작동 조건**
- 고객이 문제를 "인지하지 못함"
- 문제의 심각도가 낮음 (pain이 아니라 inconvenience)
- Total Addressable Market (TAM)이 너무 작음

**인과 메커니즘**
- 수요 부재 → 사용자 확보 어려움
- Marketing 비용 > Lifetime value of customer
- 제품은 좋을 수 있어도 시장이 없으면 사업 불가능

**경계 조건**
- 시장의 존재를 ex-post facto 판단하는 것은 쉽지만, ex-ante 예측은 어려움

**사례**
- **Google Glass** (2013~2015): 시장 준비 부족
  - 기술은 혁신적이었으나, 고객이 원하지 않음
  - 문제: AR glasses를 "매일" 쓰고 싶은 사람이 거의 없음
  - 초기 사용 사례 (enterprise): 의료, 제조 등에서만 제한적 채택
  - 소비자 시장: 이상함, 비쌈, 배터리 부족 → 수요 거의 없음

---

### F3: 목표 고객층 정의 실패 — 너무 넓거나, 잘못됨
**작동 조건**
- "모두가 우리의 고객" 마인드셋
- 특정 segment의 pain을 충분히 깊게 이해하지 못함
- 각 segment에 맞는 기능/messaging을 differentiate하지 못함

**인과 메커니즘**
- 너무 많은 고객층을 만족시키려다 → 누구도 극도로 만족시키지 못함
- Mediocre product for everyone > Great product for someone
- 경쟁사(focused segment 전문가)에게 각 segment마다 밀림

**경계 조건**
- Platform 비즈니스(network effect 있는)는 다양 segment 포용이 전략일 수 있음
- 초기 stage에서는 "narrow segment" 전문화가 필수

**사례**
- **Quibi** (2020~2021): 너무 넓은 positioning + 타이밍 오류
  - 가설: "바쁜 사람들이 10분 쇼를 통근길에 본다"
  - 문제: COVID-19 락다운 → "바쁜 사람"이 집에만 있음, TikTok과 경쟁
  - 콘텐츠: 별도 appeal 없음, 정책: 공유 불가, 스크린샷 불가
  - 결과: 2개월 만에 100만 구독자도 못 가짐, 8개월 만에 서비스 중단
  - 손실: $2B+ 투자

---

### F4: 기존 강자의 외면 또는 오판 — "우리 산업 모델이 최고"
**작동 조건**
- 기존 시장 리더가 새로운 경쟁자를 무시 or 과소평가
- 기존 business model의 high margin/switching cost에 집착
- 신기술/신 수요를 진지하게 검토하지 않음

**인과 메커니즘**
- 신입자의 성장이 눈에 띄기 시작할 때는 이미 market position 쌓여 있음
- 기존 강자의 대응은 "나중에" 시작되므로, 追격 어려움
- 초기 adopter → 주류 → 기존 강자 고객까지 넘어옴

**경계 조건**
- 기존 강자도 "혁신의 딜레마"를 극복할 수 있음 (e.g., Microsoft → cloud)
- 신입자가 항상 이기는 건 아님 (network effect, switching cost가 높으면)

**사례**
- **Blockbuster vs Netflix**:
  - 2000년: Reed Hastings (Netflix 창립자), Netflix를 $50M에 Blockbuster에 제안 → 거절
  - 이유: Blockbuster 경영진 → "온라인 렌탈은 우리 비즈니스 위협 안 함"
  - 현실: 2004년 Blockbuster peak ($5.9B 수익), 하지만 Netflix는 이미 선점
  - Blockbuster Online 출시 (2006) ← Netflix보다 6년 늦음
  - 결과: 2010년 Blockbuster 파산

---

### F5: 글로벌 확장 실패 — 로컬 경쟁자에게 패배
**작동 조건**
- 글로벌 플랫폼이 로컬 문화/규제 무시
- 로컬 경쟁자가 더 깊은 고객 이해를 가지고 있음
- network effect가 약한 카테고리 (모든 시장이 "winner-take-all"은 아님)

**인과 메커니즘**
- 글로벌 scale + 자본 ≠ 로컬 시장 지배
- 로컬 경쟁자는 규제, 문화, 고객 심리를 더 깊게 앎
- 초기 positioning에서 로컬 경쟁자가 이기면, 나중의 추격은 어려움

**경계 조건**
- 일부 카테고리는 network effect가 강해서 글로벌 승자 남음 (Google Search)
- 일부 카테고리는 로컬 특성이 강해서 로컬 승자 남음 (Ride-sharing)

**사례**
- **Uber 국제 확장**:
  - 강점: 기술, 자본, brand → US 지배
  - 약점: 로컬 규제, 운전기사 문화, 지역 경쟁자 미과소평가
  - 중국: Didi에 패배 (2016년 Uber China 자산 양도)
  - 인도: Ola와 경쟁 → 현재까지 우위 불확실
  - 원인: "세계 어디나 Uber 모델이 먹힌다" 가정 실패

---

### F6: 기술 기반 과신 — 기술이 좋아도 시장이 없으면 실패
**작동 조건**
- 혁신적 기술 개발 (R&D focused)
- 고객 수요/willingness-to-pay 미검증
- "Build it, they will come" 마인드셋

**인과 메커니즘**
- 기술 excellence ≠ product-market fit
- 기술이 고객 pain을 실제로 해결하는지 확인 필수

**경계 조건**
- 극히 드물지만, visionary 창업자의 기술 기반 추진이 성공하기도 함
- 대부분은 검증 후 기술 개발이 더 효율적

**사례**
- **Theranos**:
  - 기술: "Edison" = 몇 방울 혈액으로 250+ 검사 가능 장비
  - 문제: 기술이 작동하지 않음 (부정확한 결과, FDA는 herpes 검사만 승인)
  - 거짓 marketing: $100M 2014년 수익 claim ← 실제 $100k
  - 결과: 2018년 SEC 사기 혐의, 2022년 Holmes 유죄 판결
  - 손실: $700M+ 투자자 손실

## 진단 체크리스트

### 시장·고객 선택 전 점검 사항

**① Customer Understanding (필수)**
- [ ] 타겟 고객 segment가 명확하게 정의되었는가?
- [ ] 그 segment가 실제로 "문제"를 인식하고 있는가?
- [ ] 문제의 심각도를 정량화할 수 있는가?
- [ ] 기존 solution이 불충분한 이유가 명확한가?
- [ ] 고객 10~20명 인터뷰 실시 및 "구매 의사" 확인했는가?

**② Market Size Validation**
- [ ] Total Addressable Market (TAM)이 충분히 큰가? ($1B+ 권장)
- [ ] Serviceable Addressable Market (SAM)를 계산했는가?
- [ ] 초기 목표 시장이 현실적인가? (연 10-20%)

**③ Positioning Clarity**
- [ ] "우리는 누구를 위해, 어떤 문제를 푸는가?"를 30초 이내에 말할 수 있는가?
- [ ] 경쟁사와의 구체적 차이점이 무엇인가?
- [ ] 너무 많은 customer segment를 동시에 커버하려 하지는 않는가?

**④ Timing Check**
- [ ] 기술이 충분히 성숙했는가?
- [ ] 필수 인프라가 갖춰졌는가?
- [ ] 고객이 이 시점에 이 문제를 "절실히" 느끼고 있는가?

**⑤ Go-to-Market Strategy**
- [ ] 초기 고객 획득 경로가 명확한가?
- [ ] 초기 CAC와 LTV 비교는 충분한가?
- [ ] Word-of-mouth / community-driven growth 가능성이 있는가?

**⑥ Segment-Specific Differentiation**
- [ ] 각 segment별로 messaging/positioning이 다른가?
- [ ] 초기 단계에서는 "하나의 segment에 집중"하기로 결정했는가?

## 소상공인 시장 패턴

소상공인(매출 5억 이하, 직원 5인 이하)은 스타트업·중견과 다른 시장 역학이 작동한다. 시뮬레이션 검증(B5 시골카페, B8 무인점)에서 기존 패턴 매칭 정밀도가 낮았던 문제를 보완.

### 위치 기반 3경로

| 경로 | 시장 특성 | 핵심 전략 | BEP 기준 |
|---|---|---|---|
| **관광형** | 유동인구 높음·반복 방문 낮음. 피크 시즌 집중 | 단가 높이기 + 시즌 집중 운영. 비수기 최소 비용 | 일매출 ≥ 고정비/25일 × 1.5(시즌 변동 버퍼) |
| **주거형** | 유동인구 낮음·반복 방문 높음. 안정적 | 단골 확보(관계·습관) + 배달/온라인 보조. 월 고정 고객 50명+ 목표 | 일매출 ≥ 고정비/25일 |
| **혼합형** | 유동+주거 교차. 상권 경쟁 높음 | 관광객 유입(SNS) + 주민 고정(멤버십). 이중 가격 구조 가능 | 일매출 ≥ 고정비/25일 × 1.2 |

### 1인 운영 vs 고용 분기

| 조건 | 1인 유지 | 고용 전환 |
|---|---|---|
| 일매출 기준 | ~80만 이하 | 80만+ 지속 3개월 |
| 인건비 회수 | — | 신규 인력 투입 후 매출 20%+ 증가 예상 시만 |
| 리스크 | 번아웃·휴무 불가 | 고정비 급등·관리 부담 |

### 소상공인 대조군

| 성공 패턴 | 실패 패턴 | 성패 변수 |
|---|---|---|
| 시골 카페: 관광형 + SNS 바이럴로 방문 동기 생성 | 시골 카페: 주거형인데 관광 전략 → 유동인구 부재로 폐업 | **위치-전략 정합** |
| 무인점: 주거형 + 자판기·키오스크로 24시간 운영 | 무인점: 관광형 위치인데 무인 → 체험가치 없음 | **고객 유형-운영 모델 정합** |

## 결론: 시장·고객 선택의 핵심 원리

### 1. 검증 우선, 스케일은 나중
- 고객 100명 만족 > 고객 1000명 불만족
- 초기 고객 10-20명 인터뷰는 필수

### 2. Segment 명확성이 모든 것의 기초
- "모두가 고객" = "아무도 만족 못 함"
- 초기: 가장 단호한 segment 한 개 선택

### 3. Pain Intensity 직관보다 실측
- 정량화: effort score, pain frequency, switching cost
- "좋으면 좋겠다" ≠ "돈을 낸다"

### 4. 타이밍은 제어 불가, 준비는 제어 가능
- 기술 준비도 + 고객 인지 + 인프라를 동시 확인
- "시기상조" 판단이 가장 어려운 콜

### 5. 로컬 특성은 후발주자의 강점
- 글로벌 플레이어의 weak point = 로컬 이해 부족
- 초기 로컬 팀 구성이 향후 10년 경쟁력 결정
