# 경쟁·해자 — 어떻게 이기고 지키는가?

## 성공 패턴 라이브러리

### 1. 네트워크 효과 (Network Effects)
**작동 조건**: 양측 참여자(공급/수요), 참여자 증가가 다른 참여자의 가치를 상승
**인과 메커니즘**: 초기 사용자 → 참여자 증가 → 플랫폼 가치↑ → 자가 강화 루프
**경계 조건**: 임계 사용자 규모(10K~100K) 도달 전에 자본 고갈; 경쟁 플랫폼 등장시 다중 홈 가능

**성공 사례**:
- **Facebook (2004~2008)**: 해버드 기숙사(2004년 24시간 내 50% 가입) → 4개월 내 100만 명 → .edu 이메일 제한으로 임계값 확보 → 2006년 80% 이상 고등학교 진출 → 2006년 전국 개방 완료. 각 지역/학교별 네트워크 효과로 경쟁사 진입 불가능.
- **KakaoTalk (2010~2014)**: 한국 출시 이후 3년 만에 90% 시장 점유율 달성 → 현재 97% 독점 → 대사용자당 일 1억 건 이상 메시지 (경쟁사 진입 불가능). 반면 글로벌 진출시 현지 네트워크 부족으로 10% 이탈율.
- **Instagram (2010~2014)**: 비자 프로토콜 시각 기반 사진 공유 → 페이스북 인수(2012년 10억 달러) → 이후 10배 성장 → 10억 월간활성사용자(2018년) 달성. 모바일 네트워크 효과.

**대조 패턴**: MySpace(2003~2008) → Facebook 전환(2008년): 동일 네트워크 효과 모델이었으나, 제품 인터페이스 저하(느린 로딩, 스팸 만연, 광고 난립)로 오히려 네트워크 가치 하락 → 3년간 월 4,000만 사용자 이탈 → 2008년 4월 페이스북에 역전.

### 2. 전환비용 (Switching Costs)
**작동 조건**: 고객이 대안으로 이동시 데이터/학습/통합 비용 명시적 발생
**인과 메커니즘**: 고객이 3개월 이상 사용 → 워크플로우 내재화 → 이동 비용 > 그들의 불만
**경계 조건**: 전환비용이 지나치게 높으면 고객 분노, SaaS 경우 12개월 이상 LTV 필요

**성공 사례**:
- **Salesforce (2000~현재)**: CRM 데이터 이관 비용(평균 구현비 $25,000+), 프로세스 재학습, 워크플로우 의존도 심화 → Salesforce 자격증 사용자 증가시 이탈 저항↑ → 10개 이상 통합 구축한 기업의 이탈율 40% 감소 (최소 통합 기업 대비). 150,000개 고객 이상 보유.
- **Slack (2013~현재)**: 팀 채팅 히스토리 축적, 수천 앱 통합(Google Drive 등), 조직별 슬랙 문화 정착 → 80% 이상 임직원이 사용하는 조직의 전환 가능성 62% 감소. 높은 기업 의존도로 가격 인상 최소화 저항.
- **Apple 에코시스템 (2007~현재)**: iPhone + Mac + Watch + AirPods + iCloud + Apple Pay + 5년 보건 데이터 누적 → 다층 전환비용 (경쟁사 전환시 기기·앱·데이터 모두 손실) → 92% 고객 유지율. 평균사용자당 연매출 $140.

**대조 패턴**: WeWork (2010~2023): 임차 계약 기반 공유 오피스 → 장기 임차 채무($47B, 평균 15년), 단기 고객 리스(평균 15개월) → 기간 불일치로 역전환비용 발생(고객 이탈 시 손실 설정 고정) → 2019년 월 $219,000 손실 → 2023년 파산.

### 3. 규모의 경제 (Economies of Scale)
**작동 조건**: 고객당 비용이 고객 규모와 반비례, 인프라 공유 가능
**인과 메커니즘**: 고객 1000만 명 → 1인당 비용 1/10 → 경쟁사보다 저가 가능 → 마진율 경쟁 우위
**경계 조건**: 초기에는 손실 감수 필요; 특정 규모 이상이 되어야 우위 발동

**성공 사례**:
- **Amazon AWS (2006~현재)**: 대역폭 대량 구매 할인(벤더 비용 1/3 감소) + 커스텀 하드웨어(자제 개발로 타사 비용 절감) + 수백만 고객의 인프라 공유 → 경쟁사(Google Cloud, Azure) 대비 저가 가능 → 시장 점유율 32% (경쟁사: Google 11%, Microsoft 10%). 규모 기반 프라이싱으로 신입사 진입 불가.
- **Netflix (2007~현재)**: 수백만 구독자 데이터 기반 콘텐츠 구매 협상력 강화 → 스튜디오와의 예상 수요 기반 대량 계약 → 오리지널 콘텐츠 제작(고정비 규모화) → 282백만 구독자, 2-3% 이탈율(업계 평균 5%), 동종사 대비 재가입율 61%.

**대조 패턴**: Groupon (2008~2013): 아무 네트워크 효과·전환비용·규모 이점 없음 → 판매인 중심의 높은 운영비(비즈니스 모델과 무관) → 499개 경쟁 플랫폼 등장 → 고객 확보비 급등 → 경기 회복기 모델 와해 (불황기 모델).

### 4. 브랜드 해자 (Brand Moat)
**작동 조건**: 고객이 인식하는 품질 차별화, 일관된 경험, 신뢰 누적
**인과 메커니즘**: 브랜드 회상 → 선택 편의성 → 약간의 가격 프리미엄 용인 → 3~10% 가격 상승 가능
**경계 조건**: 브랜드는 제품 품질이 뒷받침되어야만 유지; 한 번의 신뢰 실추는 회복 어려움

**성공 사례**:
- **Apple (1997~현재)**: 설계 철학(극단적 미니멀리즘) + 하드웨어-소프트웨어 통합 + 일관된 사용자 경험 → 프리미엄 가격 용인(평균사용자당 연매출 $140, Google $55, Samsung $40) → 자본효율성(투자수익률 34%) → 92% 유지율. 브랜드만으로 3-10% 가격 프리미엄.
- **Tesla (2008~현재)**: 전기차 기술 선도 + 기업가 개인 브랜드(Elon Musk) + 혁신 신뢰 → 경쟁사 대비 30-40% 가격 프리미엄 수용. Model 3 평균가 $55K(기계식 경쟁차 대비 가격 프리미엄).

**대조 패턴**: Nokia (2003~2010): 휴대폰 시장 1위(2007년 463백만 대) → 아이폰 등장(2007년) 무시 → 5년 내 시장가치 90% 하락 → 2013년 4.4백만 대로 추락. 브랜드 신뢰가 하드웨어 기술 변화(터치스크린, 소프트웨어 에코시스템)로 무력화.

### 5. 데이터 해자 (Data Moat)
**작동 조건**: 사용자 데이터 누적, AI/ML 모델 학습이 더 좋은 제품 생성
**인과 메커니즘**: 사용자 → 데이터 → 더 나은 추천/자동화 → 더 많은 사용자 → 더 많은 데이터
**경계 조건**: 데이터의 명백한 가치를 사용자가 경험해야 함; 프라이버시 규제 위험(GDPR, CCPA)

**성공 사례**:
- **Google Search (1998~현재)**: 일일 137억 건 검색 쿼리(누적 데이터의 명백한 가치) → 알고리즘 개선 → 더 좋은 검색 결과 → 더 많은 사용자 → 더 많은 데이터의 양측 강화 루프. 90% 이상 글로벌 시장 점유율. 경쟁사(Bing, Yahoo) 데이터 기반 추격 불가능.
- **Tesla 자율주행 (2015~현재)**: 450억 마일/년(분당 100,000마일) 자율주행 데이터 → Shadow Mode(수동 개입 감지) → 자동 라벨링 및 학습 → OTA 배포 → 400만 대 차량 실시간 피드백. 경쟁사 대비 압도적 데이터 기반. 6년 누적 데이터 기반 자율주행 성능 격차 불가역.
- **Netflix 시청 데이터 (2007~현재)**: 282백만 구독자의 시청 데이터 → 콘텐츠 추천 알고리즘 → 개인화 경험 → 재구독율↑ → 2-3% 이탈율(업계 평균 5%). 데이터 기반 콘텐츠 투자 의사결정.

**대조 패턴**: Blockbuster (1985~2010): 대여 데이터 수집했으나 → 분석 기술 부재(1990년대 Fortran 코드 사용) → 알고리즘 개선 불가 → Netflix(데이터 + AI) vs Blockbuster(데이터 + 낡은 기술) 격차 벌어짐 → 모바일 스트리밍 전환 불가 → 파산.

### 6. 규제/입장 장벽 (Regulatory/Switching Barriers)
**작동 조건**: 규제 승인, 라이선스, 높은 진입 비용
**인과 메커니즘**: 경쟁사 진입 = 규제 비용/시간 3년+ → 시장 진입 장애
**경계 조건**: 규제 변화에 취약; 새로운 규제가 우리 해자를 침식할 수 있음

**성공 사례**:
- **금융 서비스 라이센스 모델**: 은행 신규 허가(3-5년 소요, 비용 수억 원 이상) → 기존 은행 독점적 지위 → 경쟁사 진입 장벽 극단적. 진출 국가마다 규제 재협상 필요(Uber 및 Airbnb 초기 규제 소송과 대조적).
- **통신 스펙트럼 독점(한국)**: SK텔레콤, KT, LGU+ 3사 스펙트럼 보유 → 신규 통신사 진입 불가능 → 요금 담합(공공요금 수준으로 관리) 가능.

**대조 패턴**: 초기 Uber/Airbnb (2009~2016): 규제 회피 모델 → 각 도시/국가별 규제 소송 → 로비 비용 > 제품개발 비용 → 규제 변화(택시협회 로비, 주택임차법 강화) → 사업 수익성 악화. 현재도 전국 규제 불확실성 높음.

## 실패 패턴 라이브러리

### 1. 해자 없는 성장 (No Moat) - Groupon
**발생 조건**: 차별화 명확하지 않음, 경쟁사도 쉽게 모방 가능
**실패 메커니즘**: 1년 성공 → 경쟁사 6개월 뒤 진입 → 기능 동등화 → 가격 경쟁 → 모두 적자
**경고 신호**: 고객이 제품 설명할 때 "경쟁사와 다를 게 뭐냐" 질문, 가격에만 민감

**구체적 사례 - Groupon (2008~2015)**:
- **성공 시기**: 2008-2011년 3년간 스타트업 성공 신화. 쿠폰 기반 소셜커머스 모델.
- **모바 부재**: 네트워크 효과 없음(상인과 소비자 간 느슨한 연결) + 전환비용 없음(경쟁사로 쉬운 이동) + 규모의 경제 불가능(판매인 중심의 높은 고정비) → 경쟁사 499개 난립.
- **실패 메커니즘**: 2008-2011 호황기(경기 침체로 소비자가 할인 추구) → 2012-2013 경기 회복기(할인 추구 감소) → 모델 붕괴. 고객 충성도 없음(Groupon 할인 후 정가 구매 거부).
- **최종 결과**: 2011년 IPO → 2013년 주가 95% 급락. 경영진 교체 반복. 현재 생존만 유지(망한 회사).
- **교훈**: "비즈니스 모델 = 판매인 고비용" + "해자 부재" = 경기 사이클에 완전히 노출된 취약한 구조.

### 2. 해자 침식 (Moat Erosion) - Kodak & Blockbuster
**발생 조건**: 기술 변화, 시장 패러다임 변화, 경쟁사의 새로운 접근
**실패 메커니즘**: 우리의 과거 장점(예: 사유 기술) → 새로운 기술 등장으로 무의미화 → 새로운 기술에 대응 느림

**구체적 사례 - Kodak (1880~2012)**:
- **과거 해자**: 필름 기술 특허 1,000개 이상 + 브랜드 (세계 사진의 80% 이상 Kodak 제품).
- **해자 침식**: Kodak이 1975년 최초 디지털 카메라 발명했으나, 관리진이 디지털을 "필름 사업의 위협"으로 판단 → 개발 투자 중단.
- **조직 장벽**: 필름 기반 화학 사업 중심의 조직 문화 + 5명의 CEO 교체 중에도 중간 관리층의 디지털 전환 저항 → 의사결정 지연.
- **경쟁사 추격**: 디지털 카메라 급속 보급(2000년대) + 스마트폰 카메라 등장(2010년대) → 시장 수요 급변.
- **최종 결과**: 2012년 파산 신청. 필름 해자는 10년 만에 완전히 무력화. 현재 회사 없음.
- **교훈**: 과거 해자를 보호하려다 새로운 해자 구축 기회 상실. 조직 문화가 기술 변화 적응을 가로막음.

**구체적 사례 - Blockbuster (1985~2010)**:
- **과거 해자**: 물리적 매장 5,000개 이상 + 고객 접근성 + "늦은 반납료" 연매출 $800M (매우 수익성 높은 사업 모델).
- **해자 침식**: Netflix의 무료 반납 정책(2000년) + 스트리밍 기술(2007년) → 완전히 다른 경쟁 축 출현.
- **경영진의 오류**: Netflix를 "장기 위협 아님"으로 판단(2000년 인수제안 $50M 거절) + Blockbuster Online(2004년 출시했으나 늦음) + 스트리밍에 투자 미흡.
- **기술 격차**: 1990년대 Fortran 코드 사용 → 데이터 분석 기술 부재 → AI 기반 추천 불가 (Netflix 대비).
- **최종 결과**: 2010년 파산 신청. 2007년 463백만 대 → 2013년 4.4백만 대로 붕괴. 현재 마지막 매장(오리건) 1개만 생존.
- **교훈**: 현재 해자의 수익성을 보호하다 새로운 파라다임 변화(DVR → 스트리밍)에 대응 실패. "하드웨어 비즈니스 → 소프트웨어 플랫폼"의 근본적 사업 모델 변화 대응 불가.

### 3. 네트워크 효과 역전 (Reverse Network Effects) - MySpace → Facebook
**발생 조건**: 네트워크 가치가 역전되는 순간, 전환비용 없는 플랫폼
**실패 메커니즘**: 플랫폼 붕괴 → 사용자 이탈 → 네트워크 가치 추락 → 급속 몰락

**구체적 사례 - MySpace (2003~2008)**:
- **성공 시기**: 2003-2007년 미국 소셜네트워크 1위. 사용자 네트워크 효과 존재.
- **네트워크 효과 역전**: 2005-2006년 UI 악화(느린 로딩, 광고 난립, 스팸 만연) → 사용자 경험 하락 → 청소년층 이탈 시작.
- **Facebook의 전략적 우위**: 2006년 전국 개방 직후(MySpace가 여전히 1위) → 깔끔한 인터페이스 + 개발자 앱 플랫폼(2007년) vs MySpace의 폐쇄적 플랫폼 → 개발 속도 차이 급벌어짐.
- **임계점 넘음**: 2006년 4월 페이스북이 MySpace를 역전 → 3년간 월 4,000만 사용자 이탈 (2006-2009).
- **최종 결과**: 2008년 4월 역전 완성. 2011년 News Corp 인수했으나 2013년 다시 매각(망한 회사).
- **교훈**: 네트워크 효과가 있더라도 제품 품질 악화 + 차용 가능성이 높으면 → 임계 순간에 급전 가능. 전환비용이 없는 플랫폼은 특히 취약.

### 4. 해자 착각 (False Moat) - Nokia
**발생 조건**: "우리의 팀이 최고다", "우리의 고객이 충성도 높다" 같은 주관적 판단
**실패 메커니즘**: 실제 차별화 없음 → 경쟁사 진입 → 고객은 즉시 이동 → "우리를 버렸다" 후회
**경고 신호**: 고객 이탈율이 예상을 초과, 경쟁사 고객 리뷰와 비슷함

**구체적 사례 - Nokia (2000~2013)**:
- **과거 해자**: 2007년 글로벌 휴대폰 시장 1위(463백만 대, 전체의 50% 이상). 브랜드 신뢰도 극도로 높음. "견고하고 오래 쓸 수 있는 핸드폰" 이미지.
- **false moat의 착각**: 경영진이 "우리의 하드웨어 기술 + 브랜드"를 절대적 해자로 판단 → iPhone 등장(2007년) 시 "터치스크린은 일시적" "우리의 물리 버튼이 더 정확"이라고 판단.
- **소프트웨어 에코시스템 격차 간과**: iPhone(App Store 2008년, 수십만 앱) vs Nokia(2012년 100,000개 앱) → 앱 다운로드 수 13백만/일 vs 구글플레이/애앱스토어 수십억/일 → 완전히 다른 차원의 에코시스템.
- **전략적 오류**: 자체 OS(Symbian) 고집 → Windows Phone 전략 변경(2011년) → 모든 경쟁사 추격 기회 상실. 경영진 의사결정 지연.
- **최종 결과**: 2007년 463백만 대 → 2013년 4.4백만 대. 5-6년 시장 점유율 95% 붕괴. 2014년 Microsoft에 사업 매각(망한 회사). 현재 회사 없음.
- **교훈**: 하드웨어 신뢰도와 현재 매출이 해자가 아님. 플랫폼 패러다임(하드웨어 → 소프트웨어 에코시스템) 변화에 대응 불가. 내부 정치(Symbian vs Windows Phone) + 조직 경직성이 실패 가속화.

### 5. 규제 리스크 (Regulatory Vulnerability)
**발생 조건**: 비즈니스가 규제 해석 변화에 극도로 의존
**실패 메커니즘**: 규제 변경 → 비용 3배 증가 또는 사업 불가 판정 → 1~2년 사이 시장 이탈

**구체적 사례 - WeWork (2010~2023)**:
- **비즈니스 모델**: 부동산 장기 임차(master lease, 평균 15년) → 단기 고객 서브렛(평균 15개월).
- **근본적 모순**: $47B 임차 채무 vs $4B 예상 임차료 수입 → 기간 불일치로 인한 극심한 리스크.
- **규제/시장 변화**: 2019년 코로나 팬데믹 이전 → 업무 공간 규제 변화(원격근무 확대) + 부동산 시장 하락 → 임차료 인상 곤란.
- **재무 붕괴**: 2018년 $1.9B 손실, 2019년 7월 월 $219,000 손실 (계속). 임차료 고정 + 수입 가변 → 손실 기하급수적 증가.
- **최종 결과**: 2023년 파산 신청. Chapter 11 진행 중. 경영진 신뢰도 추락(CEO 주당 유상감자 계획 폭로).
- **교훈**: 규제/경기 변화에 민감한 장기 고정 채무 + 변동 수입 = 극도로 취약한 구조. 임차 차익(arbitrage) 모델의 본질적 한계.

## 진단 체크리스트

1. **명확한 해자 정의 & 지속성**: 
   - 우리의 1순위 해자는 무엇인가? (네트워크 / 전환비용 / 규모 / 브랜드 / 데이터 / 규제 중 하나)
   - 3년 후에도 유효한가? (특히 기술 변화 위험도 검토)
   - 예시: Facebook이 2006년에도 지역별 네트워크 가치를 강조했으나, MySpace도 동일 모델이었음. 차이는 "제품 품질"이 미치는 영향.

2. **모방 난이도 & 경쟁사 추격 속도**:
   - 경쟁사가 우리 핵심 기능을 구현하는 데 필요한 시간은? (6개월 이상 필수)
   - Kodak은 1975년에 디지털카메라 발명했으나 40년 뒤에 망함 → "기술 보유" ≠ "해자"
   - 경쟁사 R&D 속도를 정확히 추정했는가?

3. **고객 충성도 & 전환비용 정량화**:
   - 경쟁사의 더 싼 대안이 나타났을 때, 고객의 이동 비용은? (정성적 평가 금지)
   - Salesforce: 구현비 $25K+ + 데이터 마이그레이션 + 재학습 = 수백만 원 이상
   - Apple: 에코시스템 전환 비용 = 기기 재구매(수백만 원) + 앱 재구매 + 데이터 손실
   - WeWork: 고객 이동 비용 = 0 (시장 상황 악화시 즉시 이탈)

4. **규제/기술 변화 대비 수준**:
   - 향후 3년 규제/기술 변화에서 우리의 해자가 유지될 가능성은? (<50% = 위험)
   - Blockbuster: DVR(기술) + 스트리밍(패러다임) + 온라인 구독(비즈니스 모델) = 3중 타격
   - 현재 기술 스택의 나이는? (5년 이상 = 위험)

5. **데이터/네트워크 효과의 누적 속도**:
   - 우리의 데이터나 네트워크 가치가 월 1% 이상 누적되는가?
   - Tesla: 월 5억 마일 데이터 누적 → AI 모델 재학습 주기 3~6개월 → 경쟁사는 추격 불가
   - Google: 일 137억 검색 쿼리 → 알고리즘 개선 → 더 많은 사용자 루프

6. **경쟁사 벤치마킹 & 위험도 평가**:
   - 상위 5개 경쟁사 중 우리보다 더 나은 해자를 가진 것이 3개 이상인가? (그렇다면 위험)
   - 경쟁사의 데이터 축적 속도가 우리보다 빠른가?
   - 경쟁사의 제품 개선 사이클이 우리보다 빠른가?

## 패턴 심화 분석: 시간대별 해자 유효성

### 연도별 해자 강도 변화

**초기 (0-3년): 네트워크 효과 또는 1순위 차별화**
- Facebook: 학교별 폐쇄 네트워크 (80%+ 진투율) → 다른 플랫폼 진입 불가능
- Slack: 팀 채팅 히스토리 축적 → 학습 곡선 형성
- 경쟁사 추격: 3-6개월 내 기능 동등화 가능 → 해자는 "확산 속도 차이"만 남음

**성장기 (3-7년): 전환비용 심화 & 데이터 누적**
- Salesforce: 3년차 이후 데이터 마이그레이션 비용 증가 → 이탈율 저항 강화
- Google: 검색 쿼리 누적 → 알고리즘 개선 → 경쟁사 추격 불가능 진입
- Blockbuster: 데이터(대여 기록) 있으나 AI 기술 투자 미흡 → 해자 형성 실패

**성숙기 (7년+): 다중 해자 강화 또는 침식**
- Apple: 브랜드 + 전환비용 + 데이터 → 3중 해자로 강화
- Netflix: 콘텐츠 라이브러리 + 데이터 기반 추천 + 구독 모델 → 다중 해자
- Nokia: 하드웨어만 의존 → 플랫폼 전환 실패 → 해자 급속 침식

### 해자 침식의 3가지 메커니즘

1. **기술 디스럽션 (5-10년 주기)**
   - Kodak: 필름(아날로그) → 디지털 전환 (1975년 자체 발명, 40년 뒤 파산)
   - Blockbuster: 대여점(오프라인) → 스트리밍(온라인) 전환 실패
   - Nokia: 피처폰(하드웨어) → 스마트폰(소프트웨어 에코시스템) 전환 실패

2. **경쟁사 급속 추격 (3-5년)**
   - Groupon: 이신 모델 복제 용이 → 경쟁사 499개 난립 → 경기 사이클 의존으로 모두 적자
   - 초기 SaaS: 기능 동등화 → 가격 경쟁만 남음 → 마진율 악화

3. **조직 경직성 & 의사결정 지연 (거의 모든 실패 사례)**
   - Kodak: 디지털 개발 팀 있으나 필름팀의 반발 → 중단
   - Nokia: Symbian vs Windows Phone 내부 정치 → 결정 지연 (3-5년)
   - Blockbuster: 온라인 사업(Blockbuster Online) 구축했으나 후순위 투자 → Netflix 대비 5년 뒤짐

## 축간 연결

- **f1-market로**: 니치 선택이 해자 유형 결정 (작은 니치 → 전환비용 유리, 큰 시장 → 네트워크 필수)
  - Groupon: 거대 시장(모든 소매) → 네트워크 효과 불필요한 모델 선택 → 해자 부재
  - Slack: 중소 팀(작은 니치) → 전환비용 극대화 가능
  
- **f2-bizmodel로**: 수익모델이 해자 강도 영향 (구독 = 전환비용 강화, 거래형 = 네트워크 필수)
  - Netflix 구독: 월정액 고착 + 데이터 축적 → 강한 해자
  - Groupon 거래형: 일회성 거래 → 전환비용 0 → 약한 해자
  - WeWork 임차형: 장기 고정 채무 → 역전환비용 발생 → 마이너스 해자
  
- **f4-innovation으로**: 기술 우위가 해자의 임시 보호막 (하지만 시간이 지나면 모방됨)
  - Tesla FSD: 데이터 + AI = 6년 누적 → 경쟁사 추격 어려움 (임시 해자)
  - Kodak 필름: 기술 특허 1,000개 → 디지털 변화 앞에 무력화 (기술 해자는 임시)
  
- **f5-timing으로**: 시장 진입 타이밍이 해자 구축 난이도 결정
  - Facebook: 소셜네트워크 "좌표계" 정립 후 진입 → 네트워크 해자 구축 용이
  - Groupon: 2008년 금융위기 → 할인 수요 폭증 → 성공(일시적)
  - Groupon: 2012년 경기 회복 → 할인 수요 감소 → 실패
