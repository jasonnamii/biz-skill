# 다각화 — 어떻게 사업을 넓히는가?

## 1. 성공 패턴 라이브러리

### 1.1 관련 다각화: 핵심 역량 확장 기반

**작동 조건**
- 기존 핵심 고객군·기술·채널이 명확히 정의되어 있을 때
- 신 시장의 니즈가 기존 역량의 60% 이상을 활용 가능할 때
- 신 시장이 기존 고객의 자연스러운 다음 단계일 때

**인과 메커니즘**
신 시장 진입 시 기존 역량의 대부분을 재활용하면, 신규 학습 비용이 극소화되고 기존 조직과 신사업 간의 시너지가 자동으로 발생한다. 이를 통해 신사업의 초기 실패율이 크게 감소하고, 기존 브랜드·고객·채널의 신뢰도를 신사업에 즉시 이전할 수 있다.

**경계 조건**
- "인접"의 경계가 모호해지면 역량 희석 발생 → 핵심 사업의 투자 감소로 악순환
- 신 고객 획득비가 기존 고객 대비 3배 이상이면 수익성 악화
- 신사업의 고객이 기존 고객과 상충(cannibalization)하면 전체 시장규모 축소

**성공 사례 1: Amazon — 전자책(Kindle)에서 AWS로의 유기적 확장**

2006년 AWS 출범까지 Amazon은 전자상거래 기업이었다. 하지만 자체 인프라 과부하 문제를 해결하던 과정에서, Amazon의 기술팀이 개발한 "외부 기업도 사용할 수 있는 클라우드 인프라"가 더 큰 시장임을 발견했다.

- **2006년**: AWS 공식 출범. 초기 목표는 Amazon 자체 인프라 효율화
- **2007-2010년**: 외부 스타트업·기업들의 채택 급증. S3(Simple Storage Service)가 블로그·스타트업의 표준 저장소로 부상
- **2015년**: AWS 운영이익 약 $3B(Amazon 전체 운영이익의 50% 초과)
- **2023년**: AWS 운영이익 $39.8B(Amazon 전체 영업이익의 70% 이상). Amazon의 "진정한 현금소"

**왜 성공했는가?**
- Amazon의 기존 역량(인프라 운영, 자동화, 확장성 관리)이 100% 재활용됨
- 기존 고객(전자상거래 기업)과 신 고객(SaaS, 스타트업)의 요구 기술이 동일
- 기존 조직(로지스틱·운영팀)과 신사업(클라우드팀)이 부분적으로 협력 → 독립적 사업부로 발전
- 신사업이 수익성으로 기존 사업을 능가 → 회사 전체 가치 3배 상승

**대조 사례: Kodak의 실패 — 같은 기술 자산, 조직 관성으로 망함**

Kodak은 1975년 세계 최초 디지털 카메라를 개발했다. 하지만 기존 필름 비즈니스(70% 이상의 수익)가 디지털 전환으로 위협받을 것을 우려해, 디지털 기술을 외부에 숨겼다.

- **1975-2000년**: Kodak, 1,000개 이상의 디지털 카메라 특허 보유. 하지만 내부 영상기술 사업팀은 비디오카메라/초콜릿 광고용 필름에 매진
- **2001-2005년**: Canon, Sony, Nikon이 디지털 카메라 시장 장악
- **2012년**: Kodak 파산. 보유한 1,000개 특허도 경매에 처분

**왜 실패했는가?**
- 신사업(디지털)의 이해 관계자가 기존 사업의 적이 되도록 조직 설계
- 신 고객(디지털 카메라 사용자)을 기존 고객(필름 구매자)의 위협으로 인식 → 의도적 무시
- 기술 자산은 있었으나, 경영진의 심리적 장벽(기존 사업 보호)이 더 큼

### 1.2 플랫폼 확장: 기존 고객층의 재활용

**작동 조건**
- 기존 플랫폼의 사용자 기반이 충분히 크고(최소 수십만 명) 활성도가 높을 때
- 신규 기능이 기존 기능과 자연스럽게 통합될 수 있는 기술 스택일 때
- 신 기능의 초기 채택이 기존 기능 사용을 방해하지 않을 때

**인과 메커니즘**
기존 플랫폼에 신 기능을 추가하면, 신 기능의 초기 채택 비용이 거의 0에 가깝다. 왜냐하면 기존 고객들이 이미 플랫폼에 "접근하는 습관"을 갖고 있기 때문이다. 신 기능이 기존 사용자 경험을 해치지 않는 한, 채택률은 신규 서비스 론칭보다 10배 이상 빠르다.

**경계 조건**
- 신 기능이 기존 UI/UX의 복잡도를 매우 높이면 기존 고객 이탈 위험
- 신 기능의 수익화 방식이 기존 방식과 상충하면, 두 수익원 모두 악화
- 신 기능 개발에 과다 투자되면, 기존 플랫폼 유지·개선 자원 부족

**성공 사례 1: Microsoft — Windows OS → Office Suite → Azure로의 연쇄 확장**

- **1985년**: Windows 1.0 출시. 당시 사용자 기반 매우 제한적
- **1990년**: Windows 3.0으로 대중화. 이때부터 Office(Word, Excel, Outlook) 번들 판매 시작. 기존 Windows 사용자에게 거의 추가 전환 비용 없음
- **2009년**: Azure(클라우드) 출시. 이미 Office 고객(엔터프라이즈 및 개인)이 250M 이상. Azure는 Office 데이터 저장소로 위치 → 자연스러운 확장
- **2023년**: Microsoft 시가총액 3조 달러. Azure 매출 약 $60B, Office 구독 수익도 지속 성장

**왜 성공했는가?**
- 각 단계의 신 기능이 기존 플랫폼 사용자의 "자연스러운 다음 단계" 요구
- UI는 일관성 유지, 기술 스택은 모듈식 → 복잡도 점진적 증가
- 신 기능의 초기 고객은 기존 고객(저비용), 신 고객은 신 기능 가치로 별도 획득
- 신 고객 획득 비용도 기존 고객 재활용으로 낮출 수 있음(Office 고객에게 Azure 추천 → 입소문)

**대조 사례: Quibi — 신규 기능이 기존 가정을 완전히 깸**

Quibi는 "모바일-퍼스트, 10분 미만 쇼" 플랫폼으로 2020년 $2B 자금 조성하며 출범했다. 신 기능이 아닌 완전히 다른 플랫폼이었다는 점이 핵심 실패 원인이다.

- **2020년 4월**: Quibi 론칭. 초기 다운로드 120만(첫 주)
- **2020년 6월**: 월간 활성 사용자 150만에서 급격히 증가 중단
- **2020년 10월**: Quibi 종료 발표. 거의 모든 자금 손실

**왜 실패했는가?**
- 기존 플랫폼이 없었음 → 신규 고객 획득 비용 극대화
- 신 기능의 가정(모바일, 야외, 10분)이 COVID-19로 인한 홈 스테이로 완전히 뒤집힘
- 너무 좁은 사용 사례에 과다 투자 → 고객이 집에 있을 때 쓸 수 없음 (Netflix 압승)

### 1.3 수직 통합: 공급사 리스크 제거와 마진 확보

**작동 조건**
- 핵심 공급사가 2-3개 이하로 집중되어 있을 때 (의존도 높음)
- 공급사의 마진이 전체 제조 원가의 20% 이상일 때
- 공급사와의 계약이 자주 분쟁하거나, 가격 인상이 잦을 때

**인과 메커니즘**
공급사를 직접 인수하거나 자체 생산시설을 구축하면, 공급사 마진과 협상 비용을 모두 제거할 수 있다. 이는 직접적인 원가 절감 외에도, 공급 차질 가능성을 제거해 생산 유연성과 속도를 획기적으로 높인다.

**경계 조건**
- 통합 대상이 핵심 경쟁력이 아닌 경우, 자원 낭비 → 오히려 경쟁력 약화
- 기술 변화가 빠른 산업(반도체)에서 수직 통합은 과거 노하우 보호에 불과 → 신기술에 뒤질 위험
- 통합 후 다른 고객을 동시에 서비스하려 하면, 이익 상충 발생 → 양쪽 고객 모두 불신

**성공 사례 1: Samsung — 반도체 → 스마트폰 → 가전의 수직 통합**

- **1970년대**: Samsung, 반도체 산업 진입 (정부 지원)
- **1980년대-1990년대**: 자체 D-RAM, NAND Flash 생산. 동시에 VCR, TV, 가전 제조 시작
- **2000년대**: 반도체(스스로 생산)를 스마트폰에 탑재. iPhone 이전부터 스스로 칩+디스플레이 제조
- **2010년대**: 스마트폰, 반도체, 가전의 수직 통합 완성. 각 부문의 이익률 35-40%(업계 평균 15-20%)
- **2023년**: Samsung 반도체 부문 이익률 5-8%로 하락(과잉 생산), 하지만 여전히 통합으로 기본 수익성 보호

**왜 성공했는가?**
- 핵심 공급사(반도체, 디스플레이)의 의존도 극대화 → 자체 통합의 가치 극대
- 스마트폰 성공으로 자체 칩/디스플레이 수요 증대 → 자동으로 원가 절감
- 자체 생산으로 스마트폰 설계 자유도 극대 → 차별화 가능
- 반도체 사업이 별도 수익원으로 발전 → 다른 고객(퀄컴, 애플)에도 판매

**대조 사례: Intel — 반도체 수직 통합의 한계 드러남**

Intel은 오랫동안 "칩 설계 + 자체 생산"의 완벽한 수직 통합을 자랑했다. 하지만 2020년대 들어 TSMC, Samsung 등 파운드리 기업들의 기술 추월로 완전히 역전됐다.

- **1970-2015년**: Intel, 반도체 칩셋 설계 및 자체 제조의 모노폴리. 이 통합이 핵심 경쟁력
- **2016-2020년**: Intel의 공정 기술(7nm, 5nm) 개발 지연. 하지만 내부 생산 시설 고정비 때문에 외부 파운드리 활용 어려움
- **2021년**: Intel Foundry Services 출범. "다른 회사 칩도 우리 공장에서 생산"하겠다는 전략
- **2023-2025년**: Intel Foundry, $13B 손실 누적. 주요 고객(Nvidia 등) 획득 실패. TSMC 기술(2nm) vs Intel(18A) 격차 악화

**왜 실패했는가?**
- 과거의 수직 통합이 현재의 수평 공정 모델(파운드리 비즈니스)과 충돌
- 자체 칩 설계팀과 파운드리 팀이 이익 상충 → "Intel이 자신의 공장에서 경쟁사 칩을 우선 생산하나?" 불신
- 기술 변화 속도가 생산 시설 감가상각보다 빠름 → 과거 기술에 묶인 공장의 부채화

### 1.4 크로스셀 번들: 고객 구매 흐름의 자연스러운 연장

**작동 조건**
- 고객의 구매 흐름(customer journey)에서 연속된 니즈가 명확할 때
- 신 상품이 기존 상품의 자연스러운 "다음 단계"일 때
- 신 상품의 초기 구매 의향율이 기존 고객 중 20% 이상일 때

**인과 메커니즘**
고객이 이미 Product A를 구매한 상태라면, Product B의 가치를 더 쉽게 이해한다. 왜냐하면 A와 B의 사용 맥락이 겹치기 때문이다. 이를 통해 신 상품의 마케팅 비용이 극소화되고, 고객 생애 가치(LTV)가 선형이 아닌 지수적으로 증가한다.

**경계 조건**
- 번들 구성이 "강요"가 되면, 고객은 번들 전체를 거부 → 기존 고객도 이탈
- 신 상품의 가격이 합리적이지 않으면, 기존 고객까지 신뢰 상실
- 신 상품이 기존 고객의 기존 경험을 후퇴시키면, 번들 효과 0

**성공 사례 1: Apple — Mac → iPhone → Services의 통합 번들**

- **2001-2007년**: iPod 론칭, 음악 구매자 → MP3 플레이어 사용자로의 전환 (기존 Mac 사용자 + 신규 대중)
- **2007년**: iPhone 출시. 이미 iPod 고객 3억 명이 확보되어 있었고, iPhone은 "iPod + 전화기". 초기 고객 중 50% 이상이 기존 Apple 생태계 사용자
- **2014년**: Apple Watch, AirPods 출시. 이미 iPhone 고객 4억 명 이상. Watch는 "iPhone의 팔찌 판본". 초기 구매율 35%(업계 신규 웨어러블 기준 5-10%)
- **2015년**: Apple Music, iCloud, App Store 서비스 번들화. 기존 하드웨어 고객(6억 명 이상)을 서비스 고객으로 전환
- **2023년**: Services 부문 매출 $85B(Apple 전체 매출의 25%). 매출 성장률 15-20%(하드웨어 5%). 이익률도 70% 이상(하드웨어 40%)

**왜 성공했는가?**
- 각 신 상품이 기존 상품의 "자연스러운 확장", 강요 아님
- 신 상품이 기존 고객의 기존 경험을 개선 → 역선택 가능성 극저
- 번들 가격이 합리적(개별 구매보다 저렴하거나 동일)
- 신 상품의 초기 마케팅 비용이 극저 → ROI 극대

**대조 사례: WeWork — 비관련 번들의 확장 실패 (WeLive, WeGrow)**

WeWork는 코워킹 스페이스의 성공 후, 이를 주거(WeLive)와 교육(WeGrow)으로 확장하려 했다. 번들이 아닌 완전히 다른 사업이었다는 점이 실패 원인이다.

- **2016-2018년**: WeWork, 글로벌 코워킹 공간 기업으로 성장. 사용자 70만 명, $20B 추정 가치
- **2018년**: WeLive(주거) 도입. 초기 목표 "코워킹 고객이 거주지도 이용하게". 하지만 실제 고객은 매우 제한적
- **2019년**: WeGrow(초등학교) 도입. "웹 문화를 아이들에게 가르침". 결과: 뉴욕 1개 위치만 운영
- **2019년**: SoftBank 투자 대폭 축소 후, WeWork 공동 창업자 Adam Neumann 퇴임
- **2024년**: WeWork, 파산 신청

**왜 실패했는가?**
- 주거·교육이 코워킹의 자연스러운 "다음 단계" 아님 → 완전히 다른 사업 모델, 고객, 규제
- 주거 공간의 유지비, 규제, 고객 이탈 메커니즘이 코워킹과 완전히 다름
- WeWork 브랜드의 "유연한 업무 공간"과 "집, 교육"은 고객 심리에서 불일치
- 과다 자본(SoftBank $18.5B)으로 경영 규율 손실 → 수익성 무시한 확장

### 1.5 생애주기 확장: 고객의 성장 단계별 서비스

**작동 조건**
- 기존 고객이 성장함에 따라 니즈가 명확히 진화할 때
- 각 성장 단계(SMB→중견기업→엔터프라이즈)를 서빙할 수 있는 기술·조직이 확보되어 있을 때
- 고객이 다른 기업으로 이동할 때보다, 자사의 신 제품으로 업그레이드하는 비용이 낮을 때

**인과 메커니즘**
기존 고객의 요구가 시간에 따라 단계적으로 진화한다는 것을 이해하면, 미리 그 다음 단계의 제품을 준비할 수 있다. 이를 통해 고객 전환(churn) 없이 고객 당 평균 매출(ARPU)을 선형이 아닌 지수 형태로 증가시킬 수 있다.

**경계 조건**
- 각 단계별 고객의 실제 니즈를 정확히 이해해야 함. 잘못된 가정 → 외면당함
- 각 단계 간 가격 차이가 너무 크면, 고객이 경쟁사 솔루션으로 이탈
- 신 단계의 제품이 기존 단계의 고객도 쉽게 사용 가능해야 함 (역호환성)

**성공 사례: Microsoft — Windows Home → Office Professional → Enterprise Suite의 진화**

Microsoft는 개인 사용자(Windows Home)에서 시작해, 점진적으로 소규모 사무실(Office Small Business), 중견기업(Office Professional Plus), 대기업(Microsoft 365 Enterprise)으로 확장했다.

- **1990년대**: Windows 개인용. 한 번의 라이선스 비용 ~$200
- **2000년대 초반**: Office Suite가 소규모 사무실의 표준. 연간 라이선스 비용 ~$500
- **2010년**: Office 365 클라우드 구독 도입. 엔터프라이즈 고객(1000명+ 직원) 연간 비용 $1M+
- **2020년대**: Microsoft 365(클라우드 기반 통합 솔루션). 고객당 연간 비용 평균 $200-$2000(기업 규모별 다름)
- **2024년**: Microsoft 구독 비즈니스 매출 $85B+. 각 고객 ARPU가 3배 이상 성장

**왜 성공했는가?**
- 각 단계의 신 기능이 기존 고객의 "실제 성장 단계" 반영
- 가격 차이가 합리적(단계별로 2-5배 증가) → 고객이 수용 가능
- 기존 스킬(예: Excel 사용법)이 신 단계(클라우드 Excel)에도 통용 → 학습비용 극저
- 신 단계의 초기 고객 = 기존 고객 → 마케팅 비용 극저

**실패 사례: Nokia — 단계 확장이 아닌 "카테고리 점프" 실패**

Nokia는 피처폰의 시장 리더였다. 하지만 스마트폰(iPhone)로의 진화에서, 기존 고객을 스마트폰 단계로 업그레이드시키지 못했다.

- **2000-2007년**: Nokia, 모바일 폰의 글로벌 리더(40% 시장점유). 피처폰(단순 통화+SMS)에 최적화
- **2008년**: iPhone 출시. Nokia는 스마트폰이 "틈새 시장"이라 평가, 투자 미약
- **2009-2010년**: Android 보급. Nokia의 심비안OS(piFeaturePhone 용)는 스마트폰 앱 생태계 형성 실패
- **2011년**: Nokia, Microsoft와 윈도우폰 제휴. 하지만 Microsoft 생태계는 이미 뒤쳐짐
- **2013년**: Nokia 모바일 폰 사업을 Microsoft에 매각. 파산 임박

**왜 실패했는가?**
- 신 단계(스마트폰)가 기존 단계(피처폰)의 자연스러운 진화가 아니라, "완전히 다른 제품"으로 인식
- 기존 고객(피처폰 사용자)의 니즈 변화를 과소평가 → "여전히 단순 통화만 필요"라 가정
- 신 단계로의 진화에 투자 대비 과소(R&D 2% vs 삼성 5%)
- 기존 제품군을 보호하려는 경영진의 심리적 장벽

## 2. 실패 패턴 라이브러리 (대조군)

### 2.1 비관련 다각화: 콩글로머릿 할인의 함정

**발생 조건**
- 서로 무관한 산업의 사업들을 동시에 진행할 때
- 각 사업 간에 고객·기술·채널의 겹침이 거의 없을 때
- 경영진이 "포트폴리오 분산"만을 목표로 할 때

**실패 메커니즘**
무관한 사업들을 동시 운영하면, 각 사업이 독립적으로 진화해야 하므로 경영 오버헤드가 극대화된다. 결과적으로 어떤 사업도 충분한 집중력과 자원을 받지 못해, 모든 사업이 "중간 수준"에 머무르게 된다. 이는 "콩글로머릿 할인(conglomerate discount)"을 초래해, 주가가 각 사업부의 단순 합산보다 30-50% 저평가된다.

**경고 신호**
- 사업 부문 간 협력/시너지가 거의 없음 → CEO가 모든 사업을 개별적으로 관리
- 분기마다 우선순위가 변경됨 → 각 사업 리더가 회사의 진정한 전략을 모름
- 신 사업 진입이 "현금 흐름 배분"으로만 설명됨 → 고객·기술 관점 없음

**실패 사례 1: General Electric (GE) — 금융·에너지·가전의 과도한 다각화**

GE는 130년 이상 역사 동안 전구→가전→발전소→금융 등으로 무분별하게 확장했다. CEO Jack Welch는 "다양한 포트폴리오"를 회사의 강점으로 마케팅했으나, 이는 정반대의 약점이었다.

- **2000년**: GE 시가총액 $600B(당시 최고). 하지만 구성 사업: 가전, 조명, 발전소, 의료기기, 금융(GE Capital), 미디어(NBC) → 8개 이상의 완전히 다른 산업
- **2007년**: GE Capital, GE 전체 이익의 55% 차지. 이 금융 비즈니스는 핵심 제조 사업과 무관했지만, 과도하게 부풀려짐
- **2008년 금융위기**: GE Capital의 손실 급증. 전체 GE는 "제조 회사 + 위기의 금융사"가 되어 모두 평가절하
- **2015년**: CEO Jeffrey Immelt, GE Capital 분리 (spinoff) 발표 당일 주가 11% 상승 ($28B 가치 증가)
- **2020년대**: GE 분해 발표 (3개 독립 회사로 분할). 일관성 없는 다각화로 적어도 $500B의 시가총액 손실

**왜 실패했는가?**
- 각 사업부가 서로 다른 고객, 경쟁사, 시장 사이클을 가짐 → 공통의 전략 수립 불가능
- 금융사업(GE Capital)이 성장하면서 실제 제조 사업의 투자 감소 → 경쟁력 약화
- 주가가 "GE 브랜드의 신뢰도 마이너스 + 각 사업부 가치의 할인"으로 책정됨
- 경영진 교체 때마다 전략이 바뀌어, 직원들이 회사의 미래를 예측할 수 없음

**대조 사례: Berkshire Hathaway — 다각화했으나 성공한 이유**

Berkshire Hathaway도 보험, 철도, 에너지, 소비재, 금융 등 무관한 사업들을 운영한다. 하지만 GE와 달리 성공했다. 차이는?

- **공통점**: 보험, 철도, 유틸리티, 소비재 등 서로 관계없는 사업
- **차이점 1**: CEO Warren Buffett의 "투자자 마인드셋" → 각 사업을 "독립적 투자 기회"로 평가, 운영. GE는 "CEO가 직접 운영"하려 함
- **차이점 2**: 보험 사업에서 생긴 "float"(고객이 낸 보험료를 배분 전에 보유하는 현금)을 투자 자금으로 활용 → 각 사업 간의 숨겨진 시너지
- **차이점 3**: 각 사업의 경영진에게 극도의 자율성 제공 → 각 사업이 자신의 경쟁력에 집중
- **2024년**: Berkshire Hathaway 시가총액 약 $900B, 장기 연평균 수익률 20%(S&P 500 평균 10%)

**Berkshire의 성공이 증명하는 것**: 다각화 자체가 악이 아니라, "경영 철학"이 결정 요소

### 2.2 핵심 역량 희석: 신사업이 기존 사업을 잠식

**발생 조건**
- 신규 사업이 기존 핵심 역량과 거의 관계없고, 새로운 기술·인력이 필요할 때
- 기존 사업의 최고 인재들이 신사업 팀으로 이동할 때
- 신사업에 대한 경영진의 "개인적 몰입"이 극도로 높을 때

**실패 메커니즘**
신사업이 전혀 다른 분야이면, 기존 사업의 최고 인재들이 신사업 팀으로 이동한다. 초기에는 신사업이 "미래 성장 엔진"으로 포장되지만, 신사업은 장기적 학습곡선(2-5년)을 필요로 한다. 이 기간 동안 기존 사업은 "2군 팀"으로 운영되어 경쟁력을 잃고, 신사업도 경험 부족으로 실패한다. 최악의 경우, 두 사업 모두 평탄화된다.

**경고 신호**
- CEO의 신사업 관심도가 극도로 높음(분기별 보고서 언급 빈도 3배↑)
- 기존 사업 팀에서 "컨텍스트 스위칭"이 증가(예: 마케팅 팀이 기존 제품+신사업 동시 진행)
- 신사업 임원이 기존 사업 임원보다 더 자주 경영진 회의에 등장
- 신사업의 비용(R&D, 마케팅)이 분기마다 증가, 하지만 수익 시점 명확하지 않음

**실패 사례: Yahoo — 검색 기술 희석의 결과**

Yahoo는 1990년대 검색·포탈의 리더였다. 하지만 "광고, 미디어, 이커머스" 등으로 무분별하게 확장하면서, 정작 검색 기술에 투자를 줄였다.

- **1995-2000년**: Yahoo, 검색·포탈의 글로벌 리더. 검색 기술팀에 전사 인재의 40% 집중
- **2001-2005년**: Yahoo 경영진, "검색은 이제 성숙한 시장, 미디어/광고가 미래"라고 선언. 검색팀 인력 25% 축소, 광고/미디어팀 2배 확대
- **2004년**: Google, 정확도로 Yahoo 검색을 압승. 하지만 Yahoo의 경영진은 여전히 "미디어 비전"에 몰두
- **2006년**: Yahoo, 검색 엔진을 Google에 위탁. 자신이 만든 기술 시장을 경쟁사에 넘김
- **2008-2016년**: Yahoo, 미디어/광고 전략 실패. Tumblr 인수($1.1B) → 손실 ($750M). 최종적으로 Verizon에 매각 ($4.5B, 10년 전 $125B 가치에서 96% 하락)

**왜 실패했는가?**
- 신사업(광고, 미디어)이 기존 사업(검색)의 인재를 흡수 → 검색 기술 낙후
- "검색은 과거, 미디어가 미래"라는 경영진의 가정이 잘못됨 → 과거와 미래의 투자 우선순위를 역으로 설정
- Yahoo가 할 수 있는 광고/미디어는 Google의 부가 서비스일 뿐 → 핵심 경쟁력이 아님

### 2.3 경영 자원 분산: 동시에 너무 많은 신사업

**발생 조건**
- 한 번에 3개 이상의 신사업을 동시 론칭할 때
- 각 신사업팀의 역량 배정이 full-time 60% 미만일 때
- 신사업 간의 우선순위가 명확하지 않을 때

**실패 메커니즘**
제한된 경영 자원이 여러 신사업에 분산되면, 어떤 사업도 "결정적 시점"에 필요한 자원을 받지 못한다. 각 신사업은 "거의 부족한 상태"에서 운영되어, 초기 성장 기회를 놓친다. 1년 후 "모든 사업이 실패했다"는 결론에 도달한다.

**경고 신호**
- 각 프로젝트의 리더십이 part-time으로 운영 (예: "CEO 10%, CFO 15%")
- 분기마다 우선순위가 변경됨 → 팀들이 "다음 분기 정책"을 기다림
- 신사업 예산이 분기마다 "조정" → 팀의 long-term 계획 불가능
- "급속성장"보다 "비용 절감"이 신사업의 주요 지표

**실패 사례: Google의 초기 다각화 시도 (Glass, Loon, Project Fi)**

Google은 2010년대 초반 너무 많은 신사업을 동시에 론칭했다.

- **2010년**: Google, 검색·광고에서 자유로운 수익으로 다양한 실험 시작
- **2012년**: Google Glass 프로젝트. 증강현실 안경
- **2013년**: Project Loon (기구로 인터넷 공급), Fiber (초고속 인터넷)
- **2014년**: Project Fi (모바일 가상통신사), Nest 인수 ($3.2B)
- **2014년**: Alphabet 설립, "Other Bets" 부문에 위의 모든 사업 이관
- **2015년**: Google Glass 공식 중단. Loon도 지속적 적자로 나중에 폐기
- **2020년대**: Fiber, Fi도 부분적 폐기 또는 축소. 결과: 수십억 달러 손실

**왜 실패했는가?**
- CEO가 모든 사업에 "시간 배분"하지 못함 → 각 사업 리더가 경영진 승인을 기다림
- "검색과 무관한 신사업"이 너무 많음 → 학습곡선이 5년 이상 → 초기 자금이 소진될 때까지도 수익 불명확
- Google의 광고 이익이 너무 커서, 신사업의 손실을 수년간 무시 → 의사결정 지연
- 각 신사업 팀이 "Google의 일부"라는 심리 → 독립적으로 사업화하려는 강인함 부족

**흥미로운 대조**: Google은 같은 자원으로 YouTube, Android 인수 후 오히려 성공했다. 차이는?
- YouTube, Android는 "이미 어느 정도 성숙한 제품" → 긴 학습곡선 불필요
- Google의 기존 인프라(서버, 유통)를 즉시 적용 가능
- CEO의 관심도는 낮았으나, 운영팀에 극도의 자율성 부여

### 2.4 브랜드 혼란: 신사업이 기존 브랜드 자산을 훼손

**발생 조건**
- 기존 브랜드 이미지와 완전히 다른 신사업을 같은 브랜드명으로 출시할 때
- 신사업의 타겟 고객이 기존 브랜드에 대해 전혀 다른 기대를 가질 때
- 신사업의 품질·가격·이미지가 기존 브랜드의 프리미엄 포지셔닝을 위협할 때

**실패 메커니즘**
고객은 브랜드명을 들으면 "그 브랜드가 상징하는 가치"를 즉시 연상한다. 신사업이 기존 브랜드의 이미지와 충돌하면, 신사업에 대한 신뢰가 떨어질 뿐 아니라 기존 고객도 "우리 브랜드가 이렇게 변했나?"라는 혼란에 빠진다. 결과적으로 신사업도 실패하고 기존 사업도 신뢰도 손상을 입는다.

**경고 신호**
- 신사업의 타겟 고객이 기존 고객과 "겹치지 않음"
- 마케팅 메시지가 신사업과 기존 사업 간에 충돌 (예: 럭셔리 vs 저가)
- 고객 조사에서 "이 회사가 정말 이 제품도 만드나?"라는 반응

**실패 사례: Volkswagen의 럭셔리 진출 실패**

Volkswagen은 대중적 자동차의 대표 브랜드였다. 1970년대 "국민차"로 성공한 후, 1980년대 럭셔리 세그먼트로 진출하려 했다.

- **1950-1970년**: VW Beetle, "저렴하지만 신뢰성 높은 국민차"의 대표. 브랜드 이미지 = 대중성·경제성
- **1985년**: Volkswagen Luxury Division 출범. 고급 자동차 시장 진출
- **1990년대**: Audi는 성공했으나, Volkswagen 럭셔리 라인은 시장에서 외면 (고객 혼란: "VW가 럭셔리?")
- **2000년대**: Audi를 별도 브랜드로 분리. Volkswagen은 다시 대중성 포지셔닝으로 복귀

**왜 실패했는가?**
- 고객 심리에서 "VW = 저렴하고 신뢰성 높은 차"라는 강한 이미지
- 럭셔리 세그먼트의 고객은 "프리미엄 브랜드"를 원하는데, VW 럭셔리는 "대중 브랜드의 고급 버전"으로 인식 → 하이브리드의 정체성 혼란
- 해결책: Audi를 별도 럭셔리 브랜드로 분리 (성공)

### 2.5 내부 자기잠식: 신사업이 기존 고객을 빼앗음

**발생 조건**
- 신사업이 기존 고객층의 상당 부분을 기존 사업에서 가져올 때
- 신 상품의 가격이 기존 상품보다 낮을 때 (또는 더 편리할 때)
- 신 상품이 기존 상품의 기능을 완전히 대체할 때

**실패 메커니즘**
신사업으로 인한 매출 증가가 정확히 기존 사업의 매출 감소와 같다면, 전체 시장 규모 성장이 없다. 오히려 고객이 저가 신 상품으로 이동하면서, 전체 마진은 하락한다. 최악의 경우, 고객 획득 비용은 증가(신 상품의 경쟁사 많음)하면서 마진은 하락하는 이중 악화.

**경고 신호**
- 신사업 론칭 후 3개월 내 기존 사업의 고객 전환율 상승 (>5%)
- 신사업의 고객 대부분이 기존 사업 고객임
- 신사업의 평균 주문 값(AOV)이 기존 사업보다 훨씬 낮음

**실패 사례: 소매 업체들의 온라인 전략 자기잠식**

많은 오프라인 소매 업체들이 온라인 채널을 열었을 때, 신 채널이 기존 오프라인 채널의 고객을 빼앗는 문제 발생했다.

- **1995-2005년**: 대형 마트, 백화점 등이 오프라인 중심
- **2005-2010년**: 이들이 온라인 쇼핑몰 출범
- **실제 결과**: 온라인 매출 증가 50% = 오프라인 매출 감소 40-50% + 온라인 마진 30%(오프라인 40%)
- **순 결과**: 전체 이익 = (오프라인 매출 60% × 40% 마진) + (온라인 매출 50% × 30% 마진) ≈ 전년 대비 20-25% 감소

**왜 실패했는가?**
- 온라인 채널이 신 고객을 충분히 끌어오지 못했음
- 기존 오프라인 고객이 더 저렴한 온라인으로 이동 → 자기잠식
- 오프라인 점포 운영비(임차료, 직원)는 여전히 존재 → 고정비 불변

**해결책**: 온라인과 오프라인의 세분화 (예: 온라인은 신 고객층 표적, 오프라인은 기존 VIP 고객 중심)

### 2.6 통합 실패: M&A를 통한 다각화의 함정

**발생 조건**
- 인수합병을 통해 신 사업 진입할 때
- 문화, 프로세스, 기술 스택이 크게 다를 때
- Post-Merger Integration(PMI) 계획이 불충분할 때

**실패 메커니즘**
M&A 후 통합(integration)은 "법적·재정적" 통합과 "운영·문화적" 통합의 두 가지가 필요하다. 법적 통합은 쉽지만(서류), 운영 통합은 매우 어렵다. 피인수 회사의 핵심 인재들이 "우리 회사의 미래가 명확하지 않다"고 판단하면 이직한다. 이렇게 인재가 빠지면 통합은 더욱 지연되고, 결국 피인수 회사는 "빈 껍데기"만 남는다.

**경고 신호**
- PMI 팀의 이사회 보고가 3개월 이상 지연
- 피인수 회사의 핵심 임원 이탈 소식 다수
- "문화적 맞춤"이 "재정적 이익"보다 뒤처짐

**실패 사례: Yahoo의 Tumblr 인수 ($1.1B → $0.3B 손실)**

Yahoo는 2013년 Tumblr를 $1.1B에 인수했다. 하지만 통합 과정에서 Tumblr의 핵심 인재가 대량 이탈했고, 플랫폼은 쇠퇴했다.

- **2013년**: Yahoo, Tumblr 인수 ($1.1B). 당시 Tumblr = 젊은 세대의 소셜 미디어 플랫폼
- **2014-2015년**: Yahoo, Tumblr에 광고 시스템 도입 시도. 사용자 이탈 심화 (Tumblr의 문화 = 광고 없는 순수 커뮤니티)
- **2016-2018년**: Tumblr의 핵심 개발자, 임원들 대량 이탈. "Yahoo 규정 적용" 시도로 Tumblr의 독립성 훼손
- **2019년**: Yahoo, Tumblr를 Automattic에 $3M에 매각. 손실 규모 약 $1B
- **2024년**: Tumblr는 여전히 존재하지만, 전성기 대비 90% 이상 사용자 감소

**왜 실패했는가?**
- Yahoo는 "Tumblr 인수"했으나, "Tumblr 운영"하지 못함
- Tumblr의 핵심 가치 = "기업 규율 없는 자유로운 커뮤니티"인데, Yahoo는 "광고 최적화·규정 준수"를 강제
- PMI 과정에서 Tumblr 창업자(David Karp)의 경영진 역할이 감소 → 핵심 인재의 신뢰도 저하

**성공 사례와의 대조: Microsoft의 LinkedIn 인수 ($26.2B, 2016)**

Microsoft는 LinkedIn을 인수한 후, LinkedIn의 독립성을 유지했다.

- **인수 직후**: Microsoft, LinkedIn을 "별도 비즈니스 부서"로 유지. CEO 변경 최소화
- **통합 방식**: LinkedIn의 사용자 데이터를 Microsoft 365와 연동하되, LinkedIn의 UI/정책은 변경 최소
- **2024년**: LinkedIn 매출 증가(인수 당시 $1.6B → 현재 $3B+). 사용자도 꾸준히 성장 (9억 명)
- **핵심 차이**: Yahoo는 "Tumblr를 Yahoo식으로 개편"하려 함. Microsoft는 "LinkedIn을 LinkedIn식으로 운영하되, 데이터만 공유"

## 4. 진단 체크리스트 (신사업 론칭 전 점검)

**다각화를 고려할 때, 다음의 6가지 질문에 대한 답변을 기록하세요:**

### 4.1 역량 일치도
- **Q**: 신사업이 기존 역량(기술·인프라·고객 관계)의 60% 이상을 활용하는가?
- **검증**: 신사업 팀이 기존 부서(예: 마케팅, 엔지니어링)의 도움을 얼마나 자주 받는가?
- **위험 신호**: "거의 새로운 팀이 필요함" → 관련 다각화 아님, 비관련 다각화 영역

### 4.2 고객 연속성
- **Q**: 신사업의 고객이 기존 고객과 동일하거나 인접한가?
- **검증**: 신사업 고객 중 기존 고객의 비율은? (목표: 50% 이상)
- **위험 신호**: 신사업과 기존 사업의 고객군이 완전히 다름 → 자기잠식 또는 마케팅 비용 극대화

### 4.3 채널 활용
- **Q**: 신사업이 기존 채널·마케팅·유통을 활용할 수 있는가?
- **검증**: 신 제품을 기존 채널(영업팀, 웹사이트, 파트너십)로 팔 수 있는가?
- **위험 신호**: "새로운 채널 구축 필요" → 신 사업의 초기 비용 급증

### 4.4 자기잠식 평가
- **Q**: 신사업의 초기 고객 중 기존 고객의 비율이 20% 이하인가?
- **검증**: 신사업 1년 후 기존 사업의 고객 이탈율이 5% 이상인가?
- **위험 신호**: 높은 자기잠식 → 전체 매출 성장 = 기존 사업 감소, 신사업 증가 (순증가 0)

### 4.5 경영 자원
- **Q**: 신사업팀이 full-time capacity 80% 이상 배정받았는가?
- **검증**: 신사업 리더가 "part-time"이거나, 기존 사업과 병행하는가?
- **위험 신호**: 리더가 기존 사업의 50% 이상 시간 할당 → 신사업 초기 성장 저해

### 4.6 브랜드 일관성
- **Q**: 신사업의 타겟 고객이 기존 브랜드를 긍정적으로 인식하는가?
- **검증**: 신사업 고객 조사에서 "기존 브랜드와의 관련성" 점수는?
- **위험 신호**: 신 고객이 "이 브랜드가 정말 이 분야에서도 신뢰할 수 있나?"라고 의문 → 브랜드 혼란

| 상황 | 추천 전략 | 유효 조건 | 무효 조건 | 주의사항 |

```
신사업 기회 발견
    ↓
[Q1] 기존 고객의 명확한 니즈인가?
    ├─ YES → [Q2] 기존 고객이 자동으로 구매할 확률?
    │         ├─ YES (>30%) → 플랫폼 확장 / 크로스셀
    │         └─ NO → [Q3] 신 고객을 충분히 끌어올 수 있는가?
    │                  ├─ YES + 기존 역량 60% 활용 → 관련 다각화
    │                  └─ NO → 신사업 포기 또는 파트너십 모색
    │
    └─ NO (기존 고객과 무관) → [Q4] 공급 체인 병목 해결인가?
                          ├─ YES → 수직 통합 고려
                          └─ NO → [Q5] 완전히 새로운 성장 축인가?
                               ├─ YES + CEO 헌신 + 5년 투자 가능 → 비관련 다각화 고려
                               └─ NO → 신사업 포기
```

---

## 7. 축간 연결

- **S1 (피봇)**: 다각화가 실패하면 원점으로 돌아가 피봇 재검토 필요. 특히 비관련 다각화가 실패했다면, 다시 관련 다각화로 집중
- **S2 (제휴)**: 인접 시장 진입 시, 그 시장의 주요 기업과 제휴(파트너십)로 시간과 리스크 단축 가능. 자체 진입보다 합리적 선택지
- **S4 (M&A)**: 수직 통합 또는 신 역량 필요 시 인수합병 고려. 단, PMI 계획(특히 인재 유지)이 핵심 성공 요인
- **S5 (포트폴리오)**: 다각화 포트폴리오의 성숙 단계에서 저성과 사업을 정리(divestiture)하거나, 유망 사업에 자원 재배분하는 회사의 "정해(定海)"처럼 작동

## 8. 다각화 성공의 핵심 3가지

### 1. 명확한 경계 조건 설정
- "어디까지가 우리 다각화인가?"를 처음부터 정의
- 예: Amazon은 "고객 경험 + 로지스틱" 범위 내에서만 다각화 (AWS는 기술, 물류는 로지스틱 → 경계 명확)
- 예: GE는 "이익 창출하면 되지"라는 경계 불명확 → 수십 개 무관 사업으로 산만화

### 2. 기존 조직의 에너지 유지
- 신사업이 "별도 리더십"을 받아야 함
- 기존 사업의 최고 인재를 신사업에 빼앗으면 안 됨
- 예: Microsoft는 각 사업부(Windows, Office, Azure)에 독립적 리더 배치

### 3. 조기 학습과 조정
- 신사업의 초기 18개월은 "확인" 기간, 확대 기간 아님
- 신사업이 예상 지표(고객 획득, 마진)를 놓치면 즉시 조정 또는 철수
- 예: Google의 많은 신사업(Glass, Loon)이 학습 기간을 너무 길게 가져감 (5년+) → 손실 누적

## 9. 추가 참고 사례군: 다각화의 극단과 평형

### 9.1 극단적 성공: Disney의 IP 생태계 다각화

Disney는 다각화의 "모범 사례" 중 하나다. 핵심은 "하나의 IP를 여러 채널에서 수익화"하는 방식이다.

- **영화 → 테마파크**: 영화에서 본 캐릭터를 테마파크에서 현실로 경험 (상호 강화)
- **영화 → 스트리밍**: Disney+에서 영화·드라마 독점 제공 (스트리밍 고객 확보)
- **영화 → 크루즈**: Disney Cruise Line에서 영화 세계를 테마로 한 여행 상품 (전혀 다른 고객층에 재접근)
- **통합 효과**: 2025년 Disney Experiences 부문 매출 $36.1B, 운영이익 $10B (이익률 27% = 업계 최고)

**성공 요인**
- 모든 다각화가 "하나의 IP 생태계"를 중심으로 정렬됨
- 각 채널의 타겟 고객이 다르지만, "Disney IP"라는 공통분모 존재
- 각 채널이 서로를 강화 (영화 흥행 → 테마파크 방문객 증가 → 상품 판매 증대)

---

### 9.2 극단적 실패: Uber의 무분별한 확장

Uber는 "승차 공유 → 배달 → 화물 → 자율주행 → 비행택시"로 확장을 시도했다. 대부분 부분적 실패 또는 철수했다.

- **2015년**: Uber Eats 론칭 (성공)
- **2016년**: Uber Freight 론칭 (화물 수송)
- **2017년**: Uber Elevate (비행택시) 프로젝트 (2020년 철수)
- **2019년**: Uber Autonomous Vehicles 자체 개발 중단, Aurora에 매각
- **결과**: 핵심 사업(라이드셰어)의 성장을 방해하는 자원 분산 → 여전히 적자 상태 (2023년 순이익 $1B 도달까지 장기 투자)

**실패 요인**
- 각 신사업이 기존 라이드셰어의 인프라, 브랜드를 활용하지 못함
- 자율주행·비행택시 같은 신사업은 5-10년의 R&D가 필요 → 초기 자본 소진
- CEO Travis Kalanick의 "자동화 집착"이 경영 자원 분산 초래

## 10. 시간축별 다각화 타이밍

### 10.1 다각화의 "적기"

**초기 단계 (창업 0-3년)**
- 다각화 금지. 핵심 사업에 집중
- "한 가지를 완벽하게" vs "여러 것을 중간 수준으로"
- 예외: 초기 고객의 명확한 연속 니즈 (크로스셀)

**성장 단계 (3-7년)**
- 관련 다각화 고려. 기존 고객의 신 니즈 충족
- 플랫폼 확장 검토
- 예: Netflix가 DVD → 스트리밍으로 전환 (관련 다각화, 하지만 매우 과감한 선택)

**성숙 단계 (7-15년)**
- 수직 통합, 생애주기 확장 고려
- 미래 시장 변화 대비한 선제적 다각화
- 예: Apple이 iPhone 성공 후 Watch, Services로 확장

**노후 단계 (15년+)**
- 비관련 다각화로 신 성장축 모색 (하지만 극도로 주의)
- 또는 포트폴리오 재구성 (저성과 사업 정리)
- 예: GE의 과도한 비관련 다각화 → 파산 직전

---

### 10.2 위험한 신호: 이 타이밍에는 피해야 할 다각화

**1. 핵심 사업의 성장이 둔화되었을 때**
- "신사업으로 성장 보충"이라는 생각 → 정반대의 결과
- 예: Yahoo가 검색 성장 둔화 후 미디어로 도망친 결과 → 검색 기술까지 손상

**2. 대규모 자본 조성 직후**
- "큰 자금이 들어왔으니 신사업 시작"이라는 함정
- 자금은 "핵심 사업 강화"에 우선 사용해야 함
- 예: WeWork가 SoftBank $18.5B 받은 후 무분별한 다각화

**3. 신규 CEO 취임 직후**
- "새로운 리더는 새로운 방향을 제시해야 한다"는 오해
- 처음 2년은 기존 사업의 문제점 파악, 강화에 집중
- 예: Yahoo의 여러 CEO들의 "전략 변경" 반복 → 조직의 혼란

## 11. 다각화 포기 기준

언제 다각화를 중단하고 철수해야 하는가?

### 11.1 조기 철수 기준 (18개월 이내)
- 신사업의 초기 고객 획득이 예상의 50% 미만
- 신사업의 마진이 예상의 30% 이상 악화
- 신사업으로 인한 기존 사업의 이탈율이 5% 이상

### 11.2 중기 철수 기준 (18-36개월)
- 신사업의 경로(path to profitability)가 명확하지 않음
- 신사업 팀의 리더십 변경이 2회 이상 발생
- 신사업 관련 예산이 분기마다 30% 이상 조정

### 11.3 장기 철수 기준 (36개월+)
- 신사업의 누적 손실이 예상 투자액을 30% 이상 초과
- 신사업이 기존 사업의 핵심 인재를 계속 흡수하는 상황
- 신사업의 "5년 뒤" 이익 전망이 투자액의 20배 미만

## 12. 다각화 의사결정 체크리스트 (최종 판단)

신사업 진입 결정을 내리기 전에 다음 항목들을 점검하세요:

**[필수 조건]** (모두 만족해야 진행)
- [ ] 기존 핵심 사업의 성장이 안정적으로 유지되고 있는가? (분기 성장률 > 0%)
- [ ] CEO와 핵심 경영진이 신사업에 충분한 시간을 할애할 수 있는가? (full-time 20% 이상)
- [ ] 신사업의 18-36개월 경로(roadmap)가 명확한가?
- [ ] 신사업팀의 리더(VP/Director)가 이미 확보되었거나, 외부 영입 계획이 있는가?

**[권장 조건]** (만족할수록 성공 확률 증가)
- [ ] 신사업이 기존 고객의 니즈에서 비롯된 것인가?
- [ ] 신사업에서 기존 역량(기술, 채널, 브랜드)의 60% 이상을 활용할 수 있는가?
- [ ] 신사업으로 인한 기존 고객 이탈이 10% 미만일 것으로 예상되는가?
- [ ] 신사업 진입 진입 비용이 연간 순이익의 50% 미만인가?
- [ ] 업계에서 다각화에 성공한 경쟁사 사례가 있는가?

**[위험 신호]** (하나라도 해당하면 신사업 재검토)
- [ ] CEO의 개인적 관심 → 고객/시장 기반 아님
- [ ] "현금 흐름 분산"이 주요 목표
- [ ] 신사업과 기존 사업의 고객/기술/채널이 거의 무관
- [ ] 신사업 진입을 위해 기존 사업 인재의 20% 이상 이동 예상
- [ ] 업계에서 다각화 실패 사례가 많음 (반면 성공 사례 거의 없음)
